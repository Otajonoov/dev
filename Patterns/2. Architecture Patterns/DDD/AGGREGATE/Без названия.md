
**Entity** individual obyektni modellaydi. Har bir Entity o'ziga xos noyob identifikatsiyaga ega bo'lib. Entity'ning asosiy farqlovchi xususiyati uning noyobligi, ya'ni individualligidir.

Aggregate diagrammada ikkita Aggregate turi tasvirlangan. Har bir Aggregate bir yoki bir nechta Entity'dan iborat bo'lib, ulardan biri **Aggregate Root** deb ataladi. Bundan tashqari, Aggregate'lar Value Object'larni o'z ichiga olishi mumkin. Diagrammada ko'rib turganingizdek, Value Object'lar ikkala Aggregate'da ham ishlatiladi.

**Value Object** yoki oddiygina **Value**, o'zgarmas kontseptual yaxlit tushunchani modellaydi. Model doirasida Value tushunchasi oddiy qiymatni ifodalaydi. Entity'dan farqli o'laroq, Value noyob identifikatsiyaga ega emas va Value'larning ekvivalentligi Value turida inkapsulyatsiya qilingan atributlarni taqqoslash orqali aniqlanadi. Bundan tashqari, Value Object narsa emas, balki Entity'ning miqdorini yoki o'lchamini aniqlash vositasidir.

Har bir Aggregate'ning **Root Entity**'si aggregate'da to'plangan barcha boshqa elementlarning egasi hisoblanadi. Root Entity'ning nomi Aggregate'ning kontseptual nomidir. Bu nomni Aggregate modellayotgan tushunchani to'g'ri tavsiflaydigan qilib tanlash kerak.

Aggregate'larni loyihalashning to'rtta asosiy qoidasini ko'rib chiqamiz:

1. **Aggregate chegarasida biznes invariantlarni himoya qiling**
2. **Kichik Aggregate'larni loyihalang**
3. **Boshqa Aggregate'larga faqat identifikator orqali murojaat qiling**
4. **Boshqa Aggregate'larni eventual consistency printsipi asosida yangilang**
