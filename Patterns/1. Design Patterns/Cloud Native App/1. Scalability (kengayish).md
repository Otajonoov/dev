
==Scalability== - bu serverga tushayotgan workload oshganda yoki kamayganda, tizimning resurslarini mos ravishda **kengaytirish yoki qisqartirish qobiliyati.

1. ==**Vertikal Scalability== (Scale Up/Down)*
    
    - Serverning resursini oshirish/kamaytirish (CPU, RAM).
        
    - **Afzalligi:** 
	    - Oson amalga oshiriladi.
	    - Kodni o‘zgartirish shart emas.
        
    - **Kamchiligi:**
		- Har doim resursni cheklov bo‘ladi (cheksiz kuchli server yo‘q).
		- Qimmat bo‘lishi mumkin.
        
2. **==Horizontal Scalability== (Scale Out/In)**
    - Serverlar/kontaynerlar sonini oshirish yoki kamaytirish (masalan, Kubernetes orqali)
    - Bitta kodni n ta instnsda run qlish



3. **Auto-scaling** - workload asosida avtomatik ravishda resurslar sonini oshirish va kamaytirish (Vertical/Horizontal)
4. **Database Scaling** - Sharding yoki Replication

| Atama             | Scalingga ta’siri                                                                                     |
| ----------------- | ----------------------------------------------------------------------------------------------------- |
| **Stateless**     | Tizimni oson horizontal scale qilishga imkon beradi                                                   |
| **Stateful**      | Har bir instnsga ma’lumot bog‘langan — scale qilish qiyinroq                                          |
| **Load Balancer** | Ko‘proq server ishlatganda, trafikni ularning orasida taqsimlaydi                                     |
| **Throughput**    | Qancha requestni bajarishingiz mumkinligini ko‘rsatadi. Scaling throughput’ni oshirish uchun qilinadi |
| **Latency**       | Ko‘proq user bo‘lsa, latency oshmasligi uchun scale qilinadi                                          |
| **Bottleneck**    | Scalingdan oldin uni aniqlab, yo‘qotish kerak — aks holda scaling foydasiz bo‘ladi                    |
