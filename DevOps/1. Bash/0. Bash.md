
Intrepretatsiya/Interpretator


- **Shell** - foydalanuvchi tomonidan Operatsion Tizimga (Operation System) kelgan buyruqlarni (command) qabul qilib, uning natijasini qaytaruvchi dastur
- **Bash** - Linux Shell dastur turlaridan biri va Linux Operatsion Tizimida standart (default) Shell hisoblanadi
- **Shell** faqat o’ziga tanish bo’lgan buyruqlarni qabul qilish va buyruq asosida javob qaytarish imkoniyatiga ega. Agar siz qandaydir notanish buyruq kiritsangiz, Shell sizga buyruq notanish (unknown) ekanligi haqida xabar beradi.
- **Terminal** - Shell buyruqlarini yozish imkoniyatini beruvchi “qora oyna”
- **Command Line** - matnli buyruqlarni (textual commands) yozish imkoniyatini beruvchi har qanday turdagi interfeys
- Shell script — bu shell komandalarini ketma-ket yozilgan holda faylga yozilgan dastur boʻlib, uni ishga tushirganda kompyuter har bir buyruqni ketma-ket bajaradi.
- Shell - umumiy tushuncha bo'lib, u user va OS orasidagi interface. Shell foydalanuvchining buyruqlarini olib, ularni interpretatsiya qiladi, operatsion tizimga bajarish uchun uzatadi. Shellning quyidagi turlari bor: sh, bash, zsh, fish, ksh 
- sh - Bourne shell. Standart shell deb qaraladi, deyarli barcha tizimlarda mavjud. 
- bash - Bourne Again Shell. sh ning zamonaviy, rivojlangan versiyasi. Linuxdagi default shell turi.



- Shell orqali buyruqlarni kiritish tuzilishi 3 qismdan iborat: command, option va argument
- **command** - ya’ni buyruq bu Operatsion Tizim foydalanuvchisiga maxsus amallarni yoki funksiyalarni bajarish imkoniyatini beradi
- **option** yoki **flag** - buyruq imkoniyatlaridan va maxsus xususiyatlaridan kengroq foydalanish imkoniyatini yaratadi
- **argument** - buyruqdan xususiy holatlar va fayllar uchun foydalanishda ishlatiladi. Buyruq davomidan yoziladigan keladigan barcha narsaga argument sifatida qarash mumkin, shu jumladan option ga ham


1.  **man** - buyrug’i foydalanilayotgan Linux Operatsion Tizimidagi mavjud bir buyruq va uning qo’shimcha xususiyatlari haqida to’liqroq ma'lumot olish imkoniyatini beradi
2. pwd(print work directory) - Operatsion Tizimda qaysi direktoriyada turganligi haqida to’liq ma’lumotni topish uchun ishlatiladi
3. **whoami** - Operatsion Tizimda qaysi foydalanuvchi sifatida login qilingani va foydalanilayotgani haqida ma’lumot beradi
4. cd(change directory) - Operatsion Tizimda turgan joyini ya’ni direktoriyani o’zgartirish uchun qo’llaniladi
5. **touch** - mavjud faylning yaratilgan vaqtini yangilash va yangi fayl yaratish maqsadlarida foydalaniladigan buyruq hisoblanadi
6. **mkdir**(make directory) - direktoriyalar yaratish uchun qo'llaniladigan buyruq hisoblandi
7. ls - Operatsion Tizimda turgan joyiga nisbatan qanday fayllar mavjud ekanligini bilish uchun foydalaniladigan buyruq hisoblanadi
8. **cp** - nusxalash (copy) uchun ishlatiladi
9. **rm** - fayllarni o’chirish (remove) uchun ishlatiladi. **rm -r** - rekursiv tarzda o’chirish uchun, asosan direktoriyalarni o’chirish uchun
10. **mv** - fayl joyini o’zgartirish (move) yoki fayl nomini o’zgartirish (rename) uchun ishlatiladi
11. export - global bash o'zgaruvchi yaratish uchun
12. sourse - 
13. chmod
14. read - O'zgaruvchiga qiymatni kiritish (input) orqali beradi
15. ~/.bashrc 
	1) yangi bash ochilganda ishga tushadi, faqat shu user uchun
	2) funksiya, script yoki o'zgaruvchilarni o'zida saqlaydi (GLOBAL - /etc/bash.bashrc) 
16) ~/.profile
	1) sistemaga SSH yoki konsol orqali kirilganda ishga tushadi, faqat shu user uchun
	2) funksiya, script yoki o'zgaruvchilarni o'zida saqlaydi
17) /etc/profile (yoki /etc/profile.d/*)
	1) sistemaga SSH yoki konsol orqali kirilganda ishga tushadi, hamma userlar uchun
	2) funksiya, script yoki o'zgaruvchilarni o'zida saqlaydi
18) /etc/environment
	1) sistemada hamma user va jarayonlarga tasir qiladi 
	2) faqat o'zgaruvchilarni saqlaydi (LANG="en_US.UTF-8")
19) tmux -  turli xil Linux Operatsion Sistemalar uchun terminal multiplexer hisoblanib, bir vaqtni o‘zida bir nechta terminal sessiyalaridan bir oyna orqali foydalanish imkonini beradi
	- **tmux da sessiyalar** deb terminalda ishlatilayotgan dasturlar holatiga nisbatan aytish mumkin
	- tmux sessiya terminal vaziyatini saqlaydi. Yangi terminal sessiya yaratib unga kirish uchun
20. `eval` — **buyruqni ichidan buyruq bajarish**
21. `tree` — **papkalar va fayllarni daraxt ko‘rinishida ko‘rsatadi**

Shell o'zgaruvchilarni string holatida saqlaydi, ammo ishga tushirishda o'zgaruvchilar o'z tiplariga mos holda ishlaydi:

O'zgaruchini chop etishda qo'shtirnoqdan foydalanish mumkin, ammo birtirnoqdan foydalanib bo'lmaydi

O'zgaruvchi qiymatini chaqirishdan ${} gulli qavslardan ham foydalanish mumkin, bu asosan ularni boshqa matnlardan ajratishda qo'llaniladi.

* *(Wilcard) - Bu belgining ma'nosi barcha argumentlarga mos keladi.

* \ (Escape) - Ushbu belgining asosiy vazifasi shell orqali amalga oshiriladigan \ $ " '; # ) ( * kabi maxsus belgilarni asl ma'nosidan o'zgartirish va oddiy matn sifatida ishga tushirish.

* ; (Semicolon) - Bir qatorda yozilgan bir nechta komandalarni bir biridan ajratishga xizmat qiladi, agar birinchi komanda muvaffaqqiyatsizlikka uchrasa ham ikkincha komanda ishlayveradi 

* & (Ampersand) - Komanda oxirida yozilganda uni orqa fon(background)da ishga tushiradi

* " " (Double Quotes) - Odatda komanda so'zlari orasidagi bo'sh joylarni saqlab qoladi, argumentlarni birlashtirishda ham xizmat qiladi. Qo'shtirnoq ichida $ \` va \ dan boshqa barcha belgilar o'z ma'nosini yo'qotadi. 

* ' ' (Single Quotes) - Qo'shtirnoq kabi ishlaydi. Birtirnoq ichida barcha maxsus belgilar o'z manosini yo'qotadi. 

* &&(AND) - Ikkinchi buyruq faqat birinchi buyruq muvaffaqiyatli (exit status = 0) bajarilganda ishlaydi. 

* ||(OR)- Ikkinchi buyruq faqat birinchi buyruq muvaffaqiyatsiz (exit status ≠ 0) bajarilganda ishlaydi.
