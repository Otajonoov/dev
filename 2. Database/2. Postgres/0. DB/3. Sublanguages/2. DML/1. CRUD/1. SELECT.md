
## 1. SELECT Buyrug'ining Qo'shimcha Imkoniyatlari

### LIKE - Shablon Qidirish
```sql
-- Airbus samolyotlarini topish
SELECT * FROM aircrafts WHERE model LIKE 'Airbus%';

-- % - istalgan belgilar ketma-ketligi
-- _ - bitta istalgan belgi

-- Uch harfli nomlar
SELECT * FROM airports WHERE airport_name LIKE '___';

-- NOT LIKE
SELECT * FROM aircrafts 
WHERE model NOT LIKE 'Airbus%' 
  AND model NOT LIKE 'Boeing%';
```

**Muhim:**

- `%` - 0 yoki ko'p belgiga mos keladi
- `_` - aniq bitta belgiga mos keladi
- Shablon boshida `%` bo'lsa, indeks ishlatilmaydi (sekinroq)

### Regular Expressions (Regex)
```sql
-- ~ operatori (case-sensitive)
SELECT * FROM aircrafts WHERE model ~ '^(A|Boe)';

-- !~ - inkor qilish
SELECT * FROM aircrafts WHERE model !~ '300$';

-- Belgilar:
-- ^ - boshlanish
-- $ - tugash
-- | - yoki
-- (A|B) - A yoki B
```

### BETWEEN - Oraliq
```sql
SELECT * FROM aircrafts 
WHERE range BETWEEN 3000 AND 6000;
-- 3000 va 6000 ham kiritiladi
```

### Hisoblangan Ustunlar
```sql
-- Mil'ga aylantirish
SELECT 
    model,
    range,
    round(range / 1.609, 2) AS miles
FROM aircrafts;
```

### ORDER BY - Tartiblash
```sql
-- Kamayish tartibida
SELECT * FROM aircrafts ORDER BY range DESC;

-- Ko'payish tartibida (default)
SELECT * FROM aircrafts ORDER BY range;  -- yoki ORDER BY range ASC

-- Ustun raqami bo'yicha
SELECT timezone FROM airports ORDER BY 1;
```

### DISTINCT - Takrorlanmaslik
```sql
SELECT DISTINCT timezone FROM airports;
-- Faqat unique qiymatlar
```

### LIMIT va OFFSET
```sql
-- Birinchi 3 ta
SELECT * FROM airports 
ORDER BY longitude DESC 
LIMIT 3;

-- 4-6 qatorlar (3 tani o'tkazib yuborish)
SELECT * FROM airports 
ORDER BY longitude DESC 
LIMIT 3 OFFSET 3;
```

### CASE - Shartli Ifodalar
```sql
SELECT 
    model,
    range,
    CASE 
        WHEN range < 2000 THEN 'Yaqin masofa'
        WHEN range < 5000 THEN 'O''rta masofa'
        ELSE 'Uzoq masofa'
    END AS type
FROM aircrafts;
```

## 2. JOIN - Jadvallarni Birlashtirish

### INNER JOIN (Oddiy JOIN)
```sql
-- 2 ta jadval
SELECT 
    a.model,
    s.seat_no,
    s.fare_conditions
FROM seats s
JOIN aircrafts a ON s.aircraft_code = a.aircraft_code
WHERE a.model LIKE 'Cessna%';
```

**Qanday Ishlaydi:**

1. Har bir seats qatorini har bir aircrafts qatori bilan birlashtiradi
2. `ON` shartiga mos kelmaganlarni filtrlaydi
3. `WHERE` qo'shimcha filtrlash

**Eski Sintaksis:**
```sql
-- Vergul bilan
SELECT a.model, s.seat_no
FROM seats s, aircrafts a
WHERE s.aircraft_code = a.aircraft_code
  AND a.model LIKE 'Cessna%';
```

### O'z-O'ziga JOIN (Self Join)
```sql
-- Har bir shaharni boshqa shaharlar bilan bog'lash
SELECT count(*)
FROM airports a1, airports a2
WHERE a1.city <> a2.city;

-- Yoki
SELECT count(*)
FROM airports a1
JOIN airports a2 ON a1.city <> a2.city;

-- Yoki CROSS JOIN
SELECT count(*)
FROM airports a1 CROSS JOIN airports a2
WHERE a1.city <> a2.city;
```

### LEFT OUTER JOIN - Chap Tashqi Birikma
````sql
-- Barcha samolyotlar, hatto marshruti bo'lmasa ham
SELECT 
    a.aircraft_code,
    a.model,
    count(r.aircraft_code) AS num_routes
FROM aircrafts a
LEFT OUTER JOIN routes r ON r.aircraft_code = a.aircraft_code
GROUP BY a.aircraft_code, a.model;
```

**Natija:**
```
320 | Airbus A320-200 | NULL | 0  -- marshrut yo'q
733 | Boeing 737-300  | 733  | 36 -- marshrut bor
````

### RIGHT OUTER JOIN - O'ng Tashqi Birikma
```sql
-- LEFT JOIN ni teskari yo'nalishi
SELECT ...
FROM table1
RIGHT OUTER JOIN table2 ON ...
-- table2 - bazaviy jadval
```

### FULL OUTER JOIN - To'liq Tashqi Birikma
```sql
-- Ikkala tarafdan ham mos kelmaganlar
SELECT ...
FROM table1
FULL OUTER JOIN table2 ON ...
```

### Ko'p Jadvalli JOIN
```sql
-- 5 ta jadval
SELECT 
    f.flight_no,
    t.passenger_name,
    tf.fare_conditions AS fc_ordered,
    s.fare_conditions AS fc_received
FROM boarding_passes b
JOIN ticket_flights tf ON b.ticket_no = tf.ticket_no 
                       AND b.flight_id = tf.flight_id
JOIN tickets t ON tf.ticket_no = t.ticket_no
JOIN flights f ON tf.flight_id = f.flight_id
JOIN seats s ON b.seat_no = s.seat_no 
             AND f.aircraft_code = s.aircraft_code
WHERE tf.fare_conditions <> s.fare_conditions;
```

## 3. To'plamlar Ustida Amallar

### UNION - Birlashtirish
```sql
-- Moskva YOKI Peterburg'dan
SELECT arrival_city FROM routes 
WHERE departure_city = 'Moskva'
UNION
SELECT arrival_city FROM routes 
WHERE departure_city = 'Sankt-Peterburg';

-- Dublikatlar bilan
UNION ALL
```

### INTERSECT - Kesishma
```sql
-- Moskva VA Peterburg'dan
SELECT arrival_city FROM routes 
WHERE departure_city = 'Moskva'
INTERSECT
SELECT arrival_city FROM routes 
WHERE departure_city = 'Sankt-Peterburg';
```

### EXCEPT - Ayirma
```sql
-- Peterburg'dan bor, Moskvadan yo'q
SELECT arrival_city FROM routes 
WHERE departure_city = 'Sankt-Peterburg'
EXCEPT
SELECT arrival_city FROM routes 
WHERE departure_city = 'Moskva';
```

## 4. Agregat Funksiyalar

### Asosiy Funksiyalar
```sql
-- O'rtacha
SELECT avg(total_amount) FROM bookings;

-- Maksimum
SELECT max(total_amount) FROM bookings;

-- Minimum
SELECT min(total_amount) FROM bookings;

-- Soni
SELECT count(*) FROM bookings;

-- Yig'indi
SELECT sum(total_amount) FROM bookings;
```

### GROUP BY - Guruhlash
```sql
-- Har bir shahardan marshrutlar soni
SELECT 
    departure_city,
    count(*) AS num_routes
FROM routes
WHERE departure_city = 'Moskva'
GROUP BY departure_city;
```

### HAVING - Guruhlangan Filtrlash
```sql
-- 15+ marshrut bo'lgan shaharlar
SELECT 
    departure_city,
    count(*)
FROM routes
GROUP BY departure_city
HAVING count(*) >= 15
ORDER BY count DESC;
```

**WHERE vs HAVING:**

- `WHERE` - gruppalashdan OLDIN (alohida qatorlar)
- `HAVING` - gruppalashdan KEYIN (guruhlar)

## 5. Window Functions - Oyna Funksiyalari

### Asosiy Tushuncha
```sql
SELECT 
    book_ref,
    book_date,
    count(*) OVER (
        PARTITION BY date_trunc('month', book_date)
        ORDER BY book_date
    ) AS running_count
FROM bookings
WHERE flight_id = 1;
```

**Terminlar:**

- **PARTITION BY** - bo'limlarga ajratish (GROUP BY ga o'xshash)
- **ORDER BY** - bo'lim ichida tartiblash
- **Window Frame** - hozirgi qator uchun hisoblanadigan qatorlar to'plami

### rank() - Reyting
```sql
SELECT 
    airport_name,
    city,
    latitude,
    rank() OVER (
        PARTITION BY timezone
        ORDER BY latitude DESC
    ) AS rank
FROM airports
WHERE timezone IN ('Asia/Irkutsk', 'Asia/Krasnoyarsk');
```

### WINDOW - Takrorlanishni Kamaytirish
```sql
SELECT 
    airport_name,
    latitude,
    first_value(latitude) OVER tz AS first_in_tz,
    rank() OVER tz
FROM airports
WINDOW tz AS (PARTITION BY timezone ORDER BY latitude DESC);
```

## 10. Eng Muhim Qoidalar

### Performance (Tezlik)
```sql
-- ❌ YOMON
SELECT * FROM table WHERE column LIKE '%text%';
-- Indeks ishlatilmaydi

-- ✅ YAXSHI
SELECT * FROM table WHERE column LIKE 'text%';
-- Indeks ishlatiladi
```

### NULL bilan Ishlash
```sql
-- NULL tekshirish
WHERE column IS NULL
WHERE column IS NOT NULL

-- COUNT bilan
count(*)          -- barcha qatorlar
count(column)     -- NULL emas qatorlar
```

### Alias (Taxallus)
```sql
-- Ustun taxallusi
SELECT column AS col_name

-- Jadval taxallusi
FROM table AS t
FROM table t  -- AS ixtiyoriy
```

## Xulosa

**SELECT** - eng muhim va kuchli SQL buyrug'i:

1. **Filtrla** - WHERE, HAVING
2. **Birlashtir** - JOIN
3. **Guruhlash** - GROUP BY
4. **Hisoblash** - Agregat funksiyalar
5. **Tartiblash** - ORDER BY
6. **Ichki so'rovlar** - Subqueries
7. **Oyna funksiyalari** - Window functions
8. **CTE** - WITH ... AS

**Eng Ko'p Ishlatiladigan Patternlar:**

- JOIN + WHERE + GROUP BY + ORDER BY
- Subquery in WHERE (EXISTS, IN)
- CTE + JOIN
- Window functions bilan analitika
