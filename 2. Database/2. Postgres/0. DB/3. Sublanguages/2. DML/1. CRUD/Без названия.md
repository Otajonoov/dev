
## 7.1 INSERT - Ma'lumot qo'shish

### 1. **WITH va RETURNING bilan logging qilish**

Bu usul orqali bir vaqtning o'zida ham ma'lumot qo'shish, ham log table'ga yozish mumkin:

```sql
WITH add_row AS
( INSERT INTO aircrafts_tmp
  SELECT * FROM aircrafts
  RETURNING *
)
INSERT INTO aircrafts_log
SELECT add_row.aircraft_code, add_row.model, add_row.range,
       current_timestamp, 'INSERT'
FROM add_row;
```

**Qanday ishlaydi:**

- `WITH` ichida asosiy INSERT bajariladi
- `RETURNING *` qo'shilgan qatorlarni qaytaradi
- Tashqi query esa bu ma'lumotlarni log table'ga yozadi
- Agar INSERT muvaffaqiyatsiz bo'lsa, log ham yozilmaydi

---

### 2. **ON CONFLICT - Conflict handling (eng muhim!)**

PostgreSQL'da duplicate key problem'ni ikki xil hal qilish mumkin:

#### **Variant 1: DO NOTHING - xatolik bermay o'tkazib yuborish**

```sql
INSERT INTO aircrafts_tmp
VALUES ( 'SU9', 'Sukhoi SuperJet-100', 3000 )
ON CONFLICT DO NOTHING
RETURNING *;
```

- Agar primary key yoki unique constraint buzilsa, qator qo'shilmaydi
- Lekin xatolik ham chiqmaydi
- Bu oddiy va xavfsiz usul

#### **Variant 2: DO UPDATE - UPSERT pattern (juda foydali!)**

```sql
INSERT INTO aircrafts_tmp
VALUES ( 'SU9', 'Sukhoi SuperJet', 3000 )
ON CONFLICT ON CONSTRAINT aircrafts_tmp_pkey
DO UPDATE SET model = excluded.model,
              range = excluded.range
RETURNING *;
```

**Nima bo'ladi:**

- Agar conflict bo'lsa → UPDATE bajariladi (yangi qator qo'shilmaydi)
- Agar conflict bo'lmasa → INSERT bajariladi
- `excluded` - bu qo'shmoqchi bo'lgan yangi qiymatlar
- Bu **atomic** operatsiya (UPSERT deyiladi)

**Real misolda:**

```sql
INSERT INTO users (id, name, email)
VALUES (1, 'Ali', 'ali@example.com')
ON CONFLICT (id) 
DO UPDATE SET 
    name = excluded.name,
    email = excluded.email,
    updated_at = NOW();
```

### 3. **COPY - Fayldan ma'lumot yuklash**

Katta hajmdagi ma'lumotlarni import qilish uchun eng tez usul:

```sql
-- Fayldan import
COPY aircrafts_tmp FROM '/home/postgres/aircrafts.txt';

-- Table'dan faylga export
COPY aircrafts_tmp TO '/home/postgres/aircrafts_tmp.txt'
WITH ( FORMAT csv );
```

## 7.2 UPDATE - Ma'lumotlarni yangilash

### 1. **UPDATE bilan RETURNING va logging**

```sql
WITH update_row AS
( UPDATE aircrafts_tmp
  SET range = range * 1.2
  WHERE model ~ '^Bom'
  RETURNING *
)
INSERT INTO aircrafts_log
SELECT ur.aircraft_code, ur.model, ur.range,
       current_timestamp, 'UPDATE'
FROM update_row ur;
```

---

### 2. **UPDATE bilan FROM clause (juda qulay!)**

Boshqa table'dan ma'lumot olish kerak bo'lganda:

```sql
UPDATE tickets_directions td
SET last_ticket_time = current_timestamp,
    tickets_num = tickets_num + 1
FROM flights_v f
WHERE td.departure_city = f.departure_city
  AND td.arrival_city = f.arrival_city
  AND f.flight_id = 12345;
```

**Afzalligi:** Subquery o'rniga JOIN ishlatish osonroq va tezroq.

## 7.3 DELETE - Ma'lumotlarni o'chirish

### **DELETE bilan USING clause**

Boshqa table'dan ma'lumot olgan holda o'chirish:

```sql
WITH min_ranges AS
( SELECT aircraft_code,
         rank() OVER (PARTITION BY left(model, 6) 
                      ORDER BY range) AS rank
  FROM aircrafts_tmp
  WHERE model ~ '^Airbus' OR model ~ '^Boeing'
)
DELETE FROM aircrafts_tmp a
USING min_ranges mr
WHERE a.aircraft_code = mr.aircraft_code
  AND mr.rank = 1
RETURNING *;
```

**Nima qiladi:** Har bir kompaniyaning eng kichik range'li samolyotini o'chiradi.

---

## **Eng muhim xulosalar:**

1. **RETURNING** - har doim yangilangan/qo'shilgan/o'chirilgan qatorlarni qaytaradi
2. **ON CONFLICT** - duplicate handling uchun eng yaxshi yechim
3. **WITH (CTE)** - murakkab operatsiyalarni logging bilan birlashtirish uchun
4. **FROM/USING** - UPDATE/DELETE'da boshqa table'lar bilan ishlash uchun
5. **COPY** - bulk import/export uchun eng tez usul
