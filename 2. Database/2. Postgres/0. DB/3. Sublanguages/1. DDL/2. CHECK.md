
CHECK - Qiymatlarni Tekshirish Cheklovi

```sql
CREATE TABLE progress (
    term numeric(1) CHECK (term = 1 OR term = 2),
    mark numeric(1) CHECK (mark >= 3 AND mark <= 5),
    ...
);
```


1. **Attribute-level CHECK** (ustun darajasida):

- To‘g‘ridan-to‘g‘ri ustun e’lon qilinayotgan joyda yoziladi.

- Shart faqat shu ustun qiymatiga taalluqli bo‘ladi.

```sql
CREATE TABLE progress (
  term NUMERIC(1) CHECK (term = 1 OR term = 2),
  mark NUMERIC(1) CHECK (mark >= 3 AND mark <= 5)
);
```

2. **Table-level CHECK** (jadval darajasida):

- `CONSTRAINT` so‘zi bilan alohida belgilanadi.
    
- Sintaksisi alohida, ammo funksional jihatdan bir xil.
    
- Imkon beradi: **bir nechta ustunlar bilan** ishlaydigan shartlar yozishga.

```sql
CREATE TABLE progress (
  term NUMERIC(1),
  mark NUMERIC(1),
  CONSTRAINT valid_mark CHECK (mark >= 3 AND mark <= 5)
);

CREATE TABLE progress (
  term NUMERIC(1),
  mark NUMERIC(1),
  CONSTRAINT valid_progress CHECK (
    (term = 1 AND mark BETWEEN 3 AND 5) OR
    (term = 2 AND mark BETWEEN 4 AND 5)
  )
);
```

Bu yerda `term` va `mark` ustunlari **birgalikda** tekshirilmoqda — buni faqat table-level CHECK bilan qilish mumkin.
