### 1. DDL (Data Definition Language) - Ma'lumotlarni Aniqlash Tili

**Maqsad:** Bazadagi ob'ektlarni yaratish va boshqarish

- `CREATE` - yaratish
- `ALTER` - o'zgartirish
- `DROP` - o'chirish
---

 Amaliy Maslahatla:

### 1. Nomlanish Konvensiyalari

```sql
-- Jadvallar: ko'plik, kichik harf
students, flights, bookings

-- Ustunlar: singular, snake_case
record_book, flight_no, scheduled_departure

-- Cheklovlar: ma'noli
unique_record_book
fk_progress_student
chk_positive_mark
```

### 2. CHECK va NOT NULL

```sql
-- Ikkalasi kerak!
term numeric(1) NOT NULL CHECK (term IN (1, 2))

-- NOT NULL tekshirmaydi, NULL kiritilishi mumkin
term numeric(1) CHECK (term IN (1, 2))  -- ❌ Yetarli emas
```

### 3. Bo'sh Stringlardan Himoya

```sql
-- Yomon
name text NOT NULL  -- '' o'tadi

-- Yaxshi
name text NOT NULL CHECK (name <> '')

-- Eng yaxshi
name text NOT NULL CHECK (trim(name) <> '')
```

### 4. Cheklovlarga Nom Berish

```sql
-- Yomon
CHECK (mark >= 3)  -- students_mark_check

-- Yaxshi
CONSTRAINT valid_mark CHECK (mark >= 3)
```

## Xulosa

**DDL - Ma'lumotlarni Aniqlash Tili** SQL'ning muhim qismi:

**Asosiy Tushunchalar:**

1. **DEFAULT** - standart qiymatlar
2. **CHECK** - qiymatlarni tekshirish
3. **NOT NULL** - bo'sh qiymat taqiqi
4. **UNIQUE** - yagonalik
5. **PRIMARY KEY** - asosiy kalit
6. **FOREIGN KEY** - tashqi kalit
7. **ALTER TABLE** - jadval modifikatsiyasi
8. **VIEW** - ko'rinishlar
9. **MATERIALIZED VIEW** - materiallashgan ko'rinishlar
10. **SCHEMA** - mantiqiy bo'limlar

**Referensial Yaxlitlik Strategiyalari:**

- CASCADE - tarqaladi
- RESTRICT - taqiqlaydi (darhol)
- NO ACTION - taqiqlaydi (kechiktirilgan)
- SET NULL - NULL qo'yadi
- SET DEFAULT - DEFAULT qo'yadi

**Eng Muhim:**

- Ma'lumotlar yaxlitligini ta'minlash
- To'g'ri cheklovlar qo'yish
- Tushunarli nomlar berish
- Sxemalardan foydalanish


## Xulosa

**DDL buyruqlari** - bu ma'lumotlar bazasining "skelet"ini yaratish vositalari:

1. **CREATE** - jadvallar va boshqa ob'ektlarni yaratish
2. **ALTER** - mavjud strukturani o'zgartirish
3. **DROP** - ob'ektlarni o'chirish

**Asosiy tamoyillar:**

- Ma'lumotlar butunligini ta'minlash (constraints)
- Bog'lanishlarni to'g'ri qurish (foreign keys)
- Mantiqiy nomlanish
- Indekslardan foydalanish

## DDL — Data Definition Language

**Struktura bilan ishlash** (jadval, indeks, schema yaratish/o'zgartirish)

### Asosiy DDL buyruqlari (6 ta)

1. **CREATE** - yangi obyekt yaratish
2. **ALTER** - mavjud obyektni o'zgartirish
3. **DROP** - obyektni o'chirish
4. **TRUNCATE** - table'dagi barcha ma'lumotlarni tezkor o'chirish
5. **RENAME** - obyekt nomini o'zgartirish
6. **COMMENT** - obyektga izoh qo'shish

### CREATE / ALTER obyektlari (15 ta)

1. **TABLE** - jadval
2. **VIEW** - view (virtual table)
3. **MATERIALIZED VIEW** - materialized view
4. **INDEX** - indeks
5. **UNIQUE** - unique indeks
6. **SEQUENCE** - sequence (ketma-ketlik)
7. **DATABASE** - ma'lumotlar bazasi
8. **SCHEMA** - schema (namespace)
9. **FUNCTION** - funksiya
10. **PROCEDURE** - stored procedure
11. **TRIGGER** - trigger
12. **TYPE** - custom ma'lumot tipi
13. **DOMAIN** - domain (tip cheklovi)
14. **EXTENSION** - kengaytma
15. **COLLATION** - sorting qoidalari
16. **TABLESPACE** - disk joy

### Jadval ustunlari va constraintlar (10 ta)

1. **PRIMARY KEY** - asosiy kalit
2. **FOREIGN KEY** - tashqi kalit
3. **REFERENCES** - bog'lanish ko'rsatish
4. **CHECK** - qiymat tekshiruvi
5. **DEFAULT** - default qiymat
6. **GENERATED** - avtomatik generatsiya
7. **ALWAYS** - doim generatsiya qilish
8. **BY DEFAULT** - agar qiymat berilmasa
9. **IDENTITY** - avtomatik o'suvchi ustun
10. **NOT NULL** - null bo'lmasligi kerak

### Partition & Inheritance (PostgreSQL xususiyatlari) (5 ta)

1. **PARTITION BY** - partition strategiyasi
2. **PARTITION OF** - partition yaratish
3. **ATTACH PARTITION** - partition biriktirish
4. **DETACH PARTITION** - partition ajratish
5. **INHERITS** - table inheritance

### DROP / ALTER bilan ishlatiladiganlar (5 ta)

1. **CASCADE** - bog'liq obyektlarni ham o'chirish
2. **RESTRICT** - bog'liq obyektlar bo'lsa xato berish
3. **IF EXISTS** - mavjud bo'lsa bajarish
4. **IF NOT EXISTS** - mavjud bo'lmasa bajarish
5. **OWNED** - egalik bilan bog'liq

**Jami DDL keywordlari: ~41 ta**
