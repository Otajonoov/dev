
FOREIGN KEY - Tashqi Kalit
```sql
CREATE TABLE progress (
    record_book numeric(5) REFERENCES students(record_book),
    ...
);

-- Yoki to'liq
CREATE TABLE progress (
    record_book numeric(5),
    FOREIGN KEY (record_book) 
        REFERENCES students(record_book)
);
```

### Terminologiya

- **Referencing table** (ссылающаяся) - progress
- **Referenced table** (ссылочная) - students
- **Main table** (главная) - students
- **Subordinate table** (подчиненная) - progress

---

```sql
CREATE TABLE customers (
  id SERIAL PRIMARY KEY,
  name TEXT NOT NULL
);

CREATE TABLE orders (
  id SERIAL PRIMARY KEY,
  customer_id INT,
  CONSTRAINT fk_customer FOREIGN KEY (customer_id)
    REFERENCES customers(id)
);
```

**Foreign key** constraint. Bu constraint PostgreSQL’da **referential integrity** ni ta’minlash uchun ishlatiladi.
##### Referential integrity

- Bu **ma’lumotlar o‘rtasidagi bog‘liqlik**ni buzilmasligiga kafolat beradi.    
- Masalan:
    - `orders` jadvalidagi `customer_id` — `customers` jadvalidagi mavjud `id` bo‘lishi shart.

- `FOREIGN KEY` — bu **boshqa jadvaldagi PRIMARY KEY yoki UNIQUE constraint bilan bog‘liq ustun**.
