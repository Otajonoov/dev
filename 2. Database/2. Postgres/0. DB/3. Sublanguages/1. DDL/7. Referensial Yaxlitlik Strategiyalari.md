Referensial Yaxlitlikni Boshqarish

```txt
1.  ON DELETE CASCADE - Asosiy qator o'chirilsa, bog'langan barcha qatorlar ham o'chiriladi.
   
2. ON DELETE RESTRICT - Bog'langan qatorlar bo'lsa, o'chirishga ruxsat bermaydi (darhol tekshiradi).

3. ON DELETE NO ACTION(Default) - Bog'langan qatorlar bo'lsa, o'chirishga ruxsat bermaydi (tekshirishni kechiktirishga ruxsat beradi).
- RESTRICT - darhol tekshiradi
- NO ACTION - tekshirishni transaksiya ichida kechiktirishga ruxsat beradi

4. ON DELETE SET NULL - Bog'langan qatorlarda tashqi kalit qiymati NULL ga o'zgartiriladi.
Shart: Ustunda NOT NULL bo'lmasligi kerak.

5.  ON DELETE SET DEFAULT - DEFAULT qiymat qo'yiladi.
Muhim: DEFAULT qiymat asosiy jadvalda mavjud bo'lishi shart!
CREATE TABLE progress (
    record_book numeric(5) DEFAULT 12345,
    FOREIGN KEY (record_book)
        REFERENCES students(record_book)
        ON DELETE SET DEFAULT
);

6.  ON UPDATE CASCADE
-- students jadvalida record_book 12300 → 12400 ga o'zgardi
-- progress jadvalidagi barcha 12300 avtomatik 12400 ga o'zgaradi
```
