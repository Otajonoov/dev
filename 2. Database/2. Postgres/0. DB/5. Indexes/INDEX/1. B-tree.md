### **Indeks turlari**

PostgreSQL turli xil indeks turlarini qo‘llab-quvvatlaydi:

1. **B-Tree (default)** – Eng ko‘p ishlatiladi, `=` (tenglik), `<`, `>`, `BETWEEN`, `ORDER BY` operatsiyalarini tezlashtiradi.
    
    ```sql
    sql
    CopyEdit
    CREATE INDEX idx_users_name ON users (name);
    
    ```
    
2. **Hash** – Faqat aniq moslik (`=`) bo‘yicha qidiruv uchun.
    
    ```sql
    sql
    CopyEdit
    CREATE INDEX idx_users_email_hash ON users USING hash (email);
    
    ```
    
3. **GIN (Generalized Inverted Index)** – To‘plam yoki matn qidiruvi (`jsonb`, `ARRAY`, `full-text search`) uchun.
    
    ```sql
    sql
    CopyEdit
    CREATE INDEX idx_users_tags ON users USING gin (tags);
    
    ```
    
4. **GiST (Generalized Search Tree)** – Geografik ma'lumotlar (`PostGIS`), to‘liq matn qidiruvi (`full-text search`).
    
    ```sql
    sql
    CopyEdit
    CREATE INDEX idx_gist_location ON places USING gist (location);
    
    ```
    
5. **BRIN (Block Range INdex)** – Juda katta jadvallar uchun (`time series` va `big data`).
    
    ```sql
    sql
    CopyEdit
    CREATE INDEX idx_brin ON logs USING brin (created_at);
    
    ```
    

---

### **Indeksdan foydalanishni tekshirish**

Indeks ishlayotganini `EXPLAIN ANALYZE` orqali tekshirish mumkin:

```sql
sql
CopyEdit
EXPLAIN ANALYZE SELECT * FROM users WHERE email = 'user@example.com';

```

Agar indeks ishlayotgan bo‘lsa, natijada **Index Scan** yoki **Index Only Scan** chiqadi.

---

### **Indekslarni olib tashlash**

Agar indeks kerak bo‘lmasa, uni o‘chirib tashlash mumkin:

```sql
sql
CopyEdit
DROP INDEX idx_users_email;

```

---

### **Muhim jihatlar**

- **Har bir indeks joy egallaydi**, shuning uchun faqat zarur bo‘lgan ustunlarga indeks qo‘yish kerak.
- **Indekslar yozish (INSERT, UPDATE, DELETE) tezligini sekinlashtirishi mumkin**, chunki har safar indeks ham yangilanadi.
- **To‘g‘ri indeks turini tanlash** muhim, chunki noto‘g‘ri indeks ishlatish samaradorlikni kamaytirishi mumkin.
