**Index** — это специальная структура данных, создаваемая в базе данных для ускорения поиска строк. Он действует как указатель на данные. Он хранит копии данных этих колонок в отсортированном виде, что позволяет ускорить выполнение запросов, особенно тех, которые используют условия фильтрации `WHERE`, соединении таблиц `JOIN`, сортировке `ORDER BY` и `GROUP BY`.
Однако индексы замедляют операции **INSERT, UPDATE, DELETE**, так как при изменении данных индексы также требуют обновления.

### Типы индексов
1. Clustered Index — определяет физический порядок хранения данных в таблице.  Это означает, что данные в таблице фактически хранятся в отсортированном порядке по значений индексируемого столбца .Обычно кластеризованный индекс создается на основе первичного ключа (Primary Key). 
	**Более быстрые запросы по диапазону** . Поскольку данные уже отсортированы, запросы по диапазону ( `BETWEEN`, `ORDER BY`, и т. д.) более эффективны.**Примечание:** Кластеризация в PostgreSQL не поддерживается автоматически. Если вставляются новые записи, порядок может не сохраняться, поэтому вам нужно вручную **перезапустить** команду `CLUSTER`. Если позже вы выполните кластеризацию с использованием другого индекса, предыдущий порядок кластеризации будет потерян.  PostgreSQL не имеет выделенного типа "кластеризованного индекса", как InnoDB. Вместо этого он **переупорядочивает** таблицу на основе существующего **индекса B-Tree** 
	Пример: CLUSTER users USING idx_users_age;
	Теперь PostgreSQL **перестроит таблицу в порядке `id`**.  
Но **новые записи и обновления не сохраняют порядок**, поэтому `CLUSTER` нужно периодически повторять.

2. Non-Clustered Index — это индекс, который хранит ссылки на данные, а не сами данные. В отличие от кластеризованного индекса, в таблице может быть несколько некластеризованных индексов. Они полезны для ускорения поиска по колонкам, которые не являются частью первичного ключа.

#### Как работают индексы в PostgreSQL?
3. **Ускорение поиска**
    - Индексы уменьшают количество строк, которые нужно просмотреть для выполнения запроса. Вместо того чтобы сканировать всю таблицу, PostgreSQL использует индекс для быстрого нахождения нужных строк.
    
4. **Фильтрация (`WHERE`)**
5. **Сортировка (`ORDER BY`)**
    - Индексы могут ускорить сортировку данных. Если у вас есть индекс на колонке, по которой выполняется сортировка, PostgreSQL может использовать этот индекс для быстрого получения отсортированных данных.
        
    - Например, запрос `SELECT * FROM users ORDER BY name` будет выполнен быстрее, если на колонке `name` есть индекс.
    
6. **Группировка (`GROUP BY`)**:
    - Индексы также ускоряют выполнение запросов с группировкой. Если у вас есть индекс на колонке, по которой выполняется группировка, PostgreSQL может использовать этот индекс для быстрого группирования данных.
        
    - Например, запрос `SELECT department, COUNT(*) FROM employees GROUP BY department` будет выполнен быстрее, если на колонке `department` есть индекс.

#### Затраты на поддержание индексов
- **`INSERT`**
    - При вставке новой строки в таблицу PostgreSQL должен обновить все индексы, связанные с этой таблицей.
    
- **`UPDATE`**
    - При обновлении строки PostgreSQL должен обновить все индексы, которые затрагивают изменяемые колонки. 
    
- **`DELETE`**
    - При удалении строки PostgreSQL должен удалить соответствующие записи из всех индексов.


#### HOT (Heap-Only Tuples)

PostgreSQL использует механизм **HOT (Heap-Only Tuples)** для оптимизации обновлений, которые не затрагивают индексированные колонки. Если обновление строки не изменяет значения в индексированных колонках, PostgreSQL может обойтись без обновления индексов, что ускоряет выполнение операции.


#### TID (Tuple ID)

Каждая строка в таблице PostgreSQL имеет уникальный идентификатор, называемый **TID (Tuple ID)**. TID состоит из двух частей:

- **Блок (block)**: Номер блока, в котором находится строка.
- **Смещение (offset)**: Позиция строки внутри блока.

**Зачем нужен TID?**
- Ускоряет поиск строк без индексов.
- Используется для внутренних операций PostgreSQL

#### Общий механизм работы индексов в PostgreSQL

PostgreSQL использует **единый стандартизированный интерфейс** для работы с индексами. Это означает, что все типы индексов (B-tree, Hash, GIN, GiST и др.) взаимодействуют с ядром СУБД через один и тот же интерфейс. Это упрощает добавление новых типов индексов и обеспечивает согласованность в их работе.

### Когда использовать индексы?

Индексы следует использовать в следующих случаях:
- **Большие таблицы**
- **Частые поисковые запросы**
- **Соединения таблиц** (JOIN)
- **Сортировка и группировка**: `ORDER BY` и `GROUP BY`.

### Когда не следует использовать индексы?

- **Маленькие таблицы** - индексация может быть бесполезной, так как полное сканирование (`FULL SCAN`) все равно будет быстрым
- **Частые операции `INSERT`, `UPDATE` и `DELETE`** - индексы требуют перестроения, что замедляет операции.
- **Колонки с большим количеством NULL-значений**

### План выполнения запроса

План выполнения запроса — это последовательность операций, которые СУБД выполняет для получения результата SQL-запроса. План выполнения состоит из двух основных стадий:

7. **Выборка результатов**:
    
    - На этой стадии СУБД определяет, какие строки из таблицы соответствуют условиям запроса.
        
    - Использование индексов может значительно ускорить эту стадию, так как СУБД может быстро найти нужные строки, не просматривая всю таблицу.
        
8. **Сортировка и группировка, выполнение агрегаций**:
    
    - На этой стадии СУБД сортирует результаты, группирует их и выполняет агрегатные функции (например, `SUM`, `COUNT`, `AVG`).
        
    - Индексы также могут ускорить эту стадию, особенно если данные уже отсортированы по нужному критерию.

### **Как создается индекс?**

В SQL индексы можно создавать явно или они могут создаваться автоматически.

**Пример явного создания индекса:**
```
CREATE INDEX idx_users_name ON users(name);
```
Этот индекс ускоряет поиск пользователей по имени:
```
SELECT * FROM users WHERE name = 'Quvonchbek';
```

**Пример автоматического создания индекса:**  
При создании **PRIMARY KEY** или **UNIQUE** ограничения СУБД автоматически создает индекс:
```
CREATE TABLE users (
    id SERIAL PRIMARY KEY,    -- Индекс создается автоматически
    email VARCHAR(255) UNIQUE -- Индекс также создается автоматически
);
```

**Характеристики некластеризованного индекса:**

- Таблица может иметь **несколько** некластеризованных индексов.
- Используется для быстрого поиска данных, но не влияет на физический порядок хранения.
- Может ускорить сортировку (`ORDER BY`) и группировку (`GROUP BY`).

## **Как узнать, использует ли запрос индекс?**

Можно проверить с помощью **EXPLAIN**
```
EXPLAIN ANALYZE SELECT * FROM users WHERE email = 'john@example.com';
```
Если в результате есть `Index Scan`, значит индекс используется. Если `Seq Scan`, значит происходит полное сканирование.

# 8-Bob: Indekslar

Indekslar ma'lumotlar bazasining ishlash samaradorligini oshirishga yordam beradi. PostgreSQL turli xil indeks turlarini qo'llab-quvvatlaydi. Biz faqat B-tree asosidagi indekslarni ko'rib chiqamiz. Indeks - bu jadval bilan bog'langan maxsus ma'lumot tuzilmasi bo'lib, undagi ma'lumotlar asosida yaratiladi. Indekslarni yaratishning asosiy maqsadi - ma'lumotlar bazasining ishlash samaradorligini oshirish.

## 8.1. Umumiy ma'lumot

Jadvallardagi qatorlar tartibsiz holda saqlanadi. SELECT, UPDATE va DELETE operatsiyalarini bajarish paytida DBMS kerakli qatorlarni topishi kerak. Ushbu qidiruvni tezlashtirish uchun indeks yaratiladi. Aslida u quyidagicha tashkil etilgan: jadvalning ma'lum bir qatoridagi ma'lumotlar asosida, ushbu qatorga mos keladigan indeks elementi (yozuv) qiymati shakllantiriladi. Indeks elementi va jadval qatori o'rtasidagi moslikni saqlab qolish uchun har bir elementga qatorga pointer (ko'rsatkich) joylashtiriladi. Indeks tartiblangan struktura hisoblanadi. Undagi elementlar (yozuvlar) saralangan holda saqlanadi, bu esa indeksda ma'lumotlarni qidirishni sezilarli darajada tezlashtiradi. Indeksda kerakli yozuv topilgandan so'ng, DBMS to'g'ridan-to'g'ri havola orqali jadvalning mos qatoriga o'tadi. Indeks yozuvlari jadval qatorlarining bir yoki bir nechta maydonlari qiymatlari asosida shakllantirilishi mumkin. Ushbu maydonlarning qiymatlari turli usullar bilan birlashtirilib va o'zgartirilishi mumkin. Bularning barchasi ma'lumotlar bazasi ishlab chiquvchisi tomonidan indeks yaratish paytida belgilanadi.

Jadvaldagi ma'lum qatorlarni qidirishda DBMS ning planner (rejalashtiruvchi) deb ataladigan maxsus quyi tizimi, ushbu jadval uchun WHERE shartida ko'rsatilgan ustunlar asosida yaratilgan indeks borligini tekshiradi. Agar bunday indeks mavjud bo'lsa, planner uning ushbu aniq holatda ishlatilishi maqsadga muvofiqligini baholaydi. Agar uning ishlatilishi maqsadga muvofiq bo'lsa, avval indeksda kerakli qiymatlar qidiriladi, so'ngra agar bunday qiymatlar topilsa, indeks yozuvlarida saqlangan pointer'lar yordamida jadvalga murojaat qilinadi. Shunday qilib, jadvaldagi qatorlarni to'liq ko'rib chiqish tartiblangan indeksda qidiruv va to'g'ridan-to'g'ri pointer (havola) orqali jadval qatoriga o'tish bilan almashtirilishi mumkin.

Shuni hisobga olish kerakki, indekslar ularni yaratish va jadvallardagi ma'lumotlarni yangilashda dolzarb holatda saqlash uchun ma'lum qo'shimcha xarajatlarni talab qiladi. Shuning uchun indekslardan ehtiyotkorlik bilan foydalanish kerak.

Siz jadvallar yaratganingizda ko'rgan bo'lsangiz, odatda ular uchun PRIMARY KEY yaratish nazarda tutilgan. Bunday hollarda DBMS o'zi ushbu cheklovni amalga oshirishni ta'minlaydigan indeks yaratadi. Axir, primary key mavjud bo'lganda, jadvalda bir xil qiymatlarga ega qatorlarning paydo bo'lishiga yo'l qo'yilmaydi. Indeks dublikatlarni tekshirishni juda tez bajarishga imkon beradi.

Ba'zi jadvallar uchun, masalan "Posadochnie talony" (boarding_passes), UNIQUE cheklovi ham nazarda tutilgan. Bunday hollarda ham DBMS avtomatik ravishda qiymatlarning noyobligini ta'minlash uchun ishlatilgan indeks yaratadi.

Ma'lum bir jadval uchun yaratilgan indekslarni ko'rish uchun psql utility buyrug'idan foydalanish kerak:

```

```

## 8.3. Unique indekslar

Indekslar, shuningdek, jadval qatorlaridagi atributlar qiymatlarining noyobligini ta'minlash uchun ham ishlatilishi mumkin. Bunday holatda unique indeks yaratiladi. Uni yaratish uchun quyidagi buyruq ishlatiladi:

sql

```sql
CREATE UNIQUE INDEX indeks-nomi
ON jadval-nomi ( ustun-nomi, ...);
```

Masalan, "Samolety" (aircrafts) jadvali uchun model ustuni bo'yicha unique indeks yaratamiz:

sql

```sql
CREATE UNIQUE INDEX aircrafts_unique_model_key
ON aircrafts ( model );
```

Bu holatda biz endi aircrafts jadvaliga bir xil samolyot modeli nomlariga ega qatorlarni kirita olmaymiz. Albatta, biz jadval yaratishda model ustuni uchun UNIQUE cheklovini belgilashimiz mumkin edi va u holda unique indeks avtomatik yaratilgan bo'lar edi.

Muhimi shuki, unique indekslarda NULL qiymatlarining mavjudligiga yo'l qo'yiladi, chunki ular hech qanday boshqa qiymatlar, shu jumladan bir-birlari bilan ham mos kelmaydigan deb hisoblanadi.
