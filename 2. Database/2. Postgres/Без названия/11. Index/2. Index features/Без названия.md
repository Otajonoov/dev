- **Expression index** — `CREATE INDEX ON users(lower(email))`
- **Covering index** (Index-Only Scan) — `INCLUDE` clause: `CREATE INDEX ON orders(user_id) INCLUDE (total, status)`
- **Composite index** — column order muhim! Leading column qoidasi
- **Unique index** — `NULL` qiymatlar unique hisoblanadi
- **Concurrent index** — `CREATE INDEX CONCURRENTLY` — table lock yo'q
- **Index on partition** — each partition o'z indexiga ega
