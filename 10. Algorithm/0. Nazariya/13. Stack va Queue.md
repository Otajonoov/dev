# Stack va Queue

## Stack (LIFO)

**Stack** — "oxirgi kirgan — birinchi chiqadi" (**LIFO**: Last In, First Out). Elementlar faqat bir uchidan (tepasidan) qo'shiladi va olinadi.

Tasavvur qil: tarelkalar tog'i — yangisini ustiga qo'yasan, olishda ham ustkisini olasan.

**Qo'llanishi:** funksiya chaqiruvlari (call stack), undo/redo, qavslar tekshiruvi, rekursiyani iterativga aylantirish, DFS.

```go
// Go'da stack = slice
stack := []int{}
stack = append(stack, x)          // push
top := stack[len(stack)-1]        // peek
stack = stack[:len(stack)-1]      // pop
```

## Queue (FIFO)

**Queue** — "birinchi kirgan — birinchi chiqadi" (**FIFO**: First In, First Out). Oxiriga qo'shiladi, boshidan olinadi.

Tasavvur qil: do'kondagi navbat — kim birinchi kelsa, birinchi xizmat oladi.

**Qo'llanishi:** BFS, task scheduling, message queue'lar, level-order traversal.

```go
// Go'da queue = slice (oddiy masalalar uchun yetarli)
queue := []int{}
queue = append(queue, x)   // enqueue
front := queue[0]          // peek
queue = queue[1:]          // dequeue
```

| Amal | Stack | Queue |
| ---- | ----- | ----- |
| Qo'shish | O(1) | O(1) |
| Olish | O(1) | O(1) |
| Kirish nuqtasi | faqat tepa | old va orqa |

> "Implement Queue using Stacks" masalasining g'oyasi: ikkita stack — biri kirish, biri chiqish uchun. Chiqish stack'i bo'shaganda kirish stack'ini ag'darib solasan. Har element ko'pi bilan 2 marta ko'chadi — amortized O(1).

## LeetCode'dagi asosiy stack shakllari

### 1. Qavslar (matching)

```go
// Valid Parentheses
pairs := map[byte]byte{')': '(', ']': '[', '}': '{'}
stack := []byte{}
for i := 0; i < len(s); i++ {
    c := s[i]
    if c == '(' || c == '[' || c == '{' {
        stack = append(stack, c)
    } else {
        if len(stack) == 0 || stack[len(stack)-1] != pairs[c] { return false }
        stack = stack[:len(stack)-1]
    }
}
return len(stack) == 0
```

Balans sanagichi varianti: ba'zi masalalarda (Minimum Add) stack ham shart emas — ochiq qavslar sonini sanash kifoya.

### 2. Min Stack — qo'shimcha holat saqlash

Har push'da "shu paytgacha minimal" qiymatni ham saqlaysan: yo juft `(val, minSoFar)`, yo ikkinchi parallel stack — `GetMin()` O(1) bo'ladi.

### 3. Monotonic Stack (Daily Temperatures)

Stack'da elementlar **monoton** tartibda saqlanadi. "O'ngdagi birinchi kattaroq element" tipidagi masalalar O(n²) → O(n):

```go
// Har kun uchun: nechadan keyin iliqroq kun keladi?
stack := []int{} // indekslar, temperaturasi kamayuvchi tartibda
for i, t := range temperatures {
    // yangi qiymat stack tepasidan katta bo'lsa — javobini topdik
    for len(stack) > 0 && temperatures[stack[len(stack)-1]] < t {
        j := stack[len(stack)-1]
        stack = stack[:len(stack)-1]
        res[j] = i - j
    }
    stack = append(stack, i)
}
```

### 4. Nested struktura (Decode String)

Ichma-ich strukturalarni ochish: `3[a2[c]]` — `[` ko'rganda hozirgi holatni (satr + son) stack'ga saqlab, yangisini boshlaysan; `]` da stack'dan olib birlashtirasan.

## Qachon ishlatasan? (signallar)

| Signal | Struktura |
| ------ | --------- |
| Qavslar, ochish-yopish juftliklari | stack |
| "Keyingi kattaroq/kichikroq element" | monotonic stack |
| Ichma-ich (nested) parsing | stack |
| Qavatma-qavat / eng qisqa yo'l (unweighted) | queue (BFS) |
| Oxirgi amalni bekor qilish (undo) | stack |
