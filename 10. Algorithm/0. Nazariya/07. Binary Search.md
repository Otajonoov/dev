# Binary Search (Ikkilik qidiruv)

**Binary search** — **tartiblangan** to'plamda qidiruv doirasini har qadamda **ikki barobar** qisqartiruvchi algoritm. "Bo'lib tashla va hukmronlik qil" (divide and conquer) tamoyilida ishlaydi.

Tasavvur qil: lug'atdan so'z qidirasan — o'rtasini ochasan, kerakli so'z alifboda oldinroq bo'lsa chap yarmiga, keyinroq bo'lsa o'ng yarmiga o'tasan. 1000 betlik lug'at ~10 qadamda hal.

Solishtirish uchun: **Linear search** — elementlarni birma-bir tekshirish, O(n). Tartiblanmagan ma'lumotda yagona yo'l, tartiblanganda esa binary search O(log n) beradi.

| n | Linear O(n) | Binary O(log n) |
|---|---|---|
| 1 000 | 1 000 qadam | ~10 qadam |
| 1 000 000 | 1 000 000 qadam | ~20 qadam |

## Klassik shablon

```go
func search(nums []int, target int) int {
    left, right := 0, len(nums)-1
    for left <= right {
        mid := left + (right-left)/2 // overflow'dan himoya
        switch {
        case nums[mid] == target:
            return mid
        case nums[mid] < target:
            left = mid + 1  // o'ng yarmida qidiramiz
        default:
            right = mid - 1 // chap yarmida qidiramiz
        }
    }
    return -1 // yoki `left` — qo'yish kerak bo'lgan joy (Search Insert Position)
}
```

> Loop tugaganda `left` — target'ni qo'yish mumkin bo'lgan pozitsiya. Shuning uchun Search Insert Position'da `return left`.

## Chegara qidirish (lower/upper bound)

Duplikatlar bo'lsa "birinchi/oxirgi pozitsiya" so'raladi (Find First and Last Position). Topilganda ham to'xtamay, doirani torayrtirishda davom etasan:

```go
// Birinchi >= target pozitsiya (lower bound)
func lowerBound(nums []int, target int) int {
    left, right := 0, len(nums) // e'tibor: right = len
    for left < right {
        mid := left + (right-left)/2
        if nums[mid] < target {
            left = mid + 1
        } else {
            right = mid
        }
    }
    return left
}
```

## Javob ustida binary search (eng kuchli shakl)

Ba'zan array emas, **javoblar oralig'i** ustida qidiruv qilinadi. Signal: "**minimal** shunday X toping-ki, shart bajarilsin" va shart **monoton** (X oshsa shart bajarilishi osonlashadi).

Misol — Capacity to Ship Packages: kema sig'imi qancha katta bo'lsa, muddat ichida ulgurish shuncha oson. Demak `canShip(capacity)` funksiyasi `false...false true...true` ko'rinishda — birinchi `true` ni binary search bilan topamiz:

```go
left, right := max(weights), sum(weights)
for left < right {
    mid := left + (right-left)/2
    if canShip(mid, days) {
        right = mid     // mid yaradi — kichikroq ham bo'larmikan?
    } else {
        left = mid + 1  // mid yaramadi — kattaroq kerak
    }
}
return left
```

Sqrt(x) va Valid Perfect Square ham shu shakl: javob `1..x` oralig'ida qidiriladi.

## Buzilgan monotonlik (Rotated Array, Peak)

- **Search in Rotated Sorted Array**: har qadamda kamida bitta yarim tartiblangan bo'ladi — qaysi yarim tartiblanganini aniqlab, target o'sha oraliqda-yo'qligini tekshirasan.
- **Find Peak Element**: `nums[mid] < nums[mid+1]` bo'lsa cho'qqi o'ngda, aks holda chapda (yoki mid'ning o'zi).

## Qachon ishlatasan? (signallar)

- Ma'lumot **tartiblangan** (yoki qisman tartiblangan)
- "O(log n) da toping" degan talab
- "Minimal/maksimal X, shart bajarilsin" + monoton shart → javob ustida qidiruv
- 2D matritsa qatorlari tartiblangan → matritsani 1D deb qarab qidirish mumkin

## Tez-tez uchraydigan xatolar

1. `mid := (left+right)/2` katta sonlarda overflow — `left + (right-left)/2` yoz
2. `left <= right` vs `left < right` — shablonga qarab farqlanadi, aralashtirma
3. Cheksiz loop: `left = mid` qilsang doira qisqarmasligi mumkin — har qadamda doira qisqarishiga ishonch hosil qil
