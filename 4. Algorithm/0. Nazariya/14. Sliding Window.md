# Sliding Window (Sirg'aluvchi oyna)

**Sliding window** — array/string ustida **ketma-ket (contiguous)** qismni ifodalovchi "oyna"ni yuritish texnikasi. Har qadamda oynani noldan qayta hisoblamay, **bir element qo'shib, bir element chiqarib** yangilaysan — O(n·k) yoki O(n²) → **O(n)**.

Tasavvur qil: poyezd oynasidan manzarani kuzatyapsan — oyna siljiganda butun manzara qayta chizilmaydi: bir chekkadan yangi ko'rinish kiradi, ikkinchisidan eskisi chiqadi.

Two pointers'ning maxsus holi: `left` va `right` — oynaning chegaralari, ikkalasi ham faqat oldinga yuradi.

## 1. Qat'iy o'lchamli oyna (fixed size k)

Signal: "uzunligi **k** bo'lgan subarray ichida maksimal/minimal/o'rtacha..."

```go
// Maximum Average Subarray I
sum := 0
for i := 0; i < k; i++ { sum += nums[i] } // birinchi oyna
best := sum
for i := k; i < len(nums); i++ {
    sum += nums[i] - nums[i-k] // yangi kiradi, eski chiqadi
    best = max(best, sum)
}
```

## 2. Dinamik o'lchamli oyna (eng keng tarqalgan)

Signal: "**eng uzun/qisqa** substring/subarray, shart bajarilsin". `right` doim oldinga yuradi, shart buzilganda `left` qisqartiradi:

```go
// Longest Substring Without Repeating Characters
func lengthOfLongestSubstring(s string) int {
    seen := map[byte]int{} // belgi → oxirgi ko'rilgan indeks +1
    best, left := 0, 0
    for right := 0; right < len(s); right++ {
        if idx, ok := seen[s[right]]; ok && idx > left {
            left = idx // duplikat topildi — oynani qisqartiramiz
        }
        seen[s[right]] = right + 1
        best = max(best, right-left+1)
    }
    return best
}
```

Universal qolip:

```go
left := 0
for right := 0; right < n; right++ {
    // 1. nums[right] ni oynaga QO'SH
    for /* oyna shartni buzsa */ {
        // 2. nums[left] ni oynadan CHIQAR
        left++
    }
    // 3. javobni yangila (oyna hozir yaroqli)
}
```

- **Fruit Into Baskets** — oynada ko'pi bilan 2 xil element (map size ≤ 2)
- **Longest Repeating Character Replacement** — `oynaUzunligi - engKo'pBelgi ≤ k` sharti

## 3. Oyna + chastota taqqoslash (anagram qidirish)

Signal: "s ichida p'ning **anagramlari**". O'lchami `len(p)` bo'lgan oynani yuritib, chastota massivlarini solishtirasan:

```go
// Find All Anagrams in a String
var need, window [26]int
for i := range p { need[p[i]-'a']++ }
for i := range s {
    window[s[i]-'a']++
    if i >= len(p) { window[s[i-len(p)]-'a']-- } // eski chiqadi
    if i >= len(p)-1 && window == need { res = append(res, i-len(p)+1) }
}
```

## Qachon ishlatasan? (signallar)

- **"Contiguous"** (ketma-ket) subarray/substring so'ralyapti — bu shart!
- "Eng uzun...", "eng qisqa...", "k o'lchamli...", "anagram/permutatsiya bor-yo'qligi"
- Brute-force'da har boshlanish uchun qayta hisoblash bo'lsa — oyna bilan qayta ishlatish mumkinmi deb o'yla

| | |
|---|---|
| Time | O(n) — har element ko'pi bilan 2 marta (kirish/chiqish) |
| Space | O(k) yoki O(alifbo) — oyna holati (map/array) |

> **Eslatma:** elementlar tartibsiz tanlansa (contiguous bo'lmasa) — bu sliding window EMAS, boshqa texnika (sort, hash, DP) kerak.
