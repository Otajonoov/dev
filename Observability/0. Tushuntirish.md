**OpenTelemetry** — bu kuzatuv (observability) ma’lumotlarini (ya’ni **tracing**, **metrics** va **logs**) **yig‘ish**, **qayta ishlash** va **eksport qilish** uchun ishlab chiqilgan ochiq standartlar to‘plami.

OpenTelemetry ishlab chiqaruvchidan (vendor) **mustaqil** bo‘lib, uni istalgan backend xizmatlar (Jaeger, Prometheus, Grafana, Honeycomb va boshqalar) bilan birga ishlatish mumkin.

---

##### 1. **Spetsifikatsiyalar (Specifications)**
API, SDK va ma’lumot formatlari uchun **standart talablar**ni belgilaydi.

---

##### 2.  **API**
Ilovaga OpenTelemetry qo‘shish uchun kerakli **interfeyslar**. Siz `tracer`, `meter` va `logger` kabi obyektlarni shu orqali chaqirasiz.

---
##### 3. **SDK**

API va eksport qiluvchilar o‘rtasidagi **asosiy ishchi modul**:
- Tracelarni kuzatadi
- Ma’lumotlarni yig‘adi
- Sampling (tanlash), filtering (filtrlash) va batching (guruhlash) ishlarini qiladi
---
##### 6.  **Kuzatuv tushunchalari (Tracing Basics)**

| Tushuncha  | Tavsifi                                                                     |
| ---------- | --------------------------------------------------------------------------- |
| `Trace`    | Bir ish oqimi (masalan, foydalanuvchi so‘rovi) bo‘ylab barcha operatsiyalar |
| `Span`     | Har bir alohida operatsiya (masalan, HTTP so‘rovi yoki SQL query)           |
| `Context`  | So‘rov identifikatori, Trace ID, Span ID’ni o‘zida olib yuradi              |
| `Resource` | Ilovani tavsiflovchi ma’lumot (xizmat nomi, pod ID va h.k.)                 |
