# Graph

**Graph** — bir-biriga bog'langan node'lar (vertex) kolleksiyasi. Bog'lanishlar **edge** deyiladi.

- **Directed** (yo'nalishli) — edge bir tomonlama: Twitter'dagi follow
- **Undirected** (yo'nalishsiz) — ikki tomonlama: Facebook'dagi do'stlik
- **Weighted** (vaznli) — har edge'ning narxi/masofasi bor: xaritadagi yo'llar

## Atamalar

- **Vertex** (node) — tugun
- **Path** — ikki node orasidagi yo'l; uzunligi = edge'lar soni
- **Cycle** — boshlangan node'ida tugaydigan yopiq yo'l
- **Degree** — node'ning bog'lanishlari soni
- **DAG** — directed acyclic graph (yo'nalishli, cycle'siz)

## Saqlash: Adjacency List

```go
// Eng qulay ko'rinish — qo'shnilar ro'yxati
graph := map[int][]int{
    0: {1, 2},
    1: {0, 3},
    2: {0, 3},
    3: {1, 2, 4},
    4: {},
}
```

LeetCode'da graph ko'pincha **yashirin** bo'ladi: 2D grid (har katak — node, qo'shni kataklar — edge), kurslar ro'yxati, uchish reyslari.

## BFS — qavatma-qavat (queue)

Boshlang'ich node'dan **eng qisqa yo'l** (unweighted) kafolati bilan tarqaladi:

```go
func bfs(graph map[int][]int, start int) {
    visited := map[int]bool{start: true}
    queue := []int{start}
    for len(queue) > 0 {
        node := queue[0]
        queue = queue[1:]
        for _, nb := range graph[node] {
            if !visited[nb] {
                visited[nb] = true
                queue = append(queue, nb)
            }
        }
    }
}
```

## DFS — chuqurlikka (rekursiya/stack)

Bir yo'ldan oxirigacha kirib boradi, keyin qaytib boshqasini sinaydi:

```go
func dfs(graph map[int][]int, node int, visited map[int]bool) {
    visited[node] = true
    for _, nb := range graph[node] {
        if !visited[nb] {
            dfs(graph, nb, visited)
        }
    }
}
```

| | BFS | DFS |
|---|---|---|
| Struktura | queue | stack / rekursiya |
| Kafolat | eng qisqa yo'l (unweighted) | to'liq qamrab olish |
| Ishlatilishi | shortest path, level'lar | komponentlar, cycle, topologik sort |

**Number of Islands** — grid DFS/BFS klassikasi: har ko'rilmagan '1' katakdan qidiruvni boshlab, butun orolni "cho'ktirasan" (visited belgilaysan), orollar sonini sanaysan.

> Grid masalalarida yo'nalishlar: `dirs := [][2]int{{-1,0},{1,0},{0,-1},{0,1}}`

## Topological Sort (Course Schedule)

DAG'da "oldin A, keyin B" tartibini topish. **Kahn algoritmi** (BFS ko'rinishi):
1. Har node'ning **in-degree**'sini (kiruvchi edge'lar) hisobla
2. In-degree 0 bo'lganlarni queue'ga sol
3. Queue'dan olib, qo'shnilarining in-degree'sini kamaytir; 0 bo'lganlari queue'ga
4. Hamma node chiqsa — tartib bor; chiqmasa — **cycle bor** (Course Schedule javobi shu!)

## Dijkstra — vaznli graphda eng qisqa yo'l

BFS'ning vaznli versiyasi: oddiy queue o'rniga **min-heap** — har safar "hozircha eng arzon" node qayta ishlanadi.

- Ishlaydi: manfiy bo'lmagan vaznlarda
- Ishlamaydi: **manfiy edge**'larda — unda Bellman-Ford kerak
- Murakkablik: O((V + E) log V)

**Cheapest Flights Within K Stops** — Dijkstra varianti, lekin qadam cheklovi borligi uchun Bellman-Ford uslubidagi k-iteratsiyali yechim quloyroq.

## MST — Minimum Spanning Tree (Min Cost to Connect All Points)

Barcha node'larni **minimal umumiy narx** bilan bog'lash:
- **Prim**: bitta daraxtdan boshlab, har safar eng arzon chiquvchi edge qo'shiladi (min-heap bilan)
- **Kruskal**: edge'larni narx bo'yicha tartiblab, cycle hosil qilmaydiganlarini qo'shib borish (Union-Find bilan)

## Qachon qaysi algoritm? (signallar)

| Signal | Algoritm |
| ------ | -------- |
| Eng qisqa yo'l, vaznsiz | BFS |
| Komponentlar / orollar soni | DFS yoki BFS |
| Bajarilish tartibi, cycle borligi (prerequisite'lar) | Topological sort |
| Eng qisqa yo'l, vaznli (manfiy emas) | Dijkstra |
| Hammani eng arzon bog'lash | MST (Prim/Kruskal) |
