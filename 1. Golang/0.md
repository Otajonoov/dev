Простота Go — это не ограничение, а сила. Отсекая ненужную сложность, Go упрощает управление кодом и позволяет разработчикам сосредоточиться на решении реальных проблем, а не на борьбе с языком.

Хорошо спроектированный язык — это не только о том, чтобы делать вещи легкими — это о том, чтобы усложнить совершение неправильных действий.

Go поощряет дизайн struct, где нулевое значение валидно и значимо. Это уменьшает необходимость в конструкторах и минимизирует риск ошибок, вызванных неинициализированными полями.

```
Generics позволяют писать функции и структуры данных без привязки их к конкретным типам. Это делает ваш код более гибким, легче поддерживаемым и устраняет избыточность — вы пишете его один раз, и он работает для любого совместимого типа.

Эта гибкость сопровождается добавленной сложностью, особенно в том, как теперь работает система типов Go:

- **Параметры типов**: Функции, типы и методы теперь могут принимать параметры типов, вводя новый уровень сложности как для компилятора, так и для разработчика.
- **Выведение типов**: Компилятор должен определить, какие типы использовать, основываясь на том, как вызываются generic-функции и типы, добавляя сложность во время компиляции.
- **Constraint'ы и границы типов**: Generics не принимают любой тип — вы должны использовать constraint'ы (comparable, any, constraints.Ordered и т.д.) для определения разрешенных типов. Эти constraint'ы часто полагаются на интерфейсы, которые служат контрактами для валидных типов.
```


```
Go, C и языки, такие как Rust, являются компилируемыми. Код, который вы пишете, не запускается напрямую. Вместо этого он проходит через процесс компиляции, переводя его в машинный код — набор низкоуровневых инструкций, которые CPU может выполнять напрямую.

В отличие от этого, программы, написанные на интерпретируемых языках, выполняются во время выполнения с помощью интерпретатора, который обрабатывает код динамически, а не компилирует его заранее. (Некоторые интерпретаторы выполняют высокоуровневый код напрямую, в то время как другие запускают bytecode — промежуточное представление исходного кода.)

Ключевое преимущество компилируемых языков — скорость. Поскольку код уже переведен в машинный код до выполнения, нет накладных расходов во время выполнения. Никакого интерпретатора, никаких задержек — только чистая производительность.

Однако компромисс — портативность. Скомпилированная программа привязана к архитектуре, для которой она была создана. Чтобы запустить ее на другой платформе, вам нужно будет перекомпилировать ее для этой конкретной системы.

Статическая типизация — еще одна основная особенность. Go и C статически типизированы, что означает, что типы переменных определяются во время компиляции. Вы объявляете тип заранее, и он не меняется. Эта стабильность позволяет компилятору оптимизировать более агрессивно, генерируя более быстрый, более эффективный машинный код.

```

**Иллюстрация 1. Быстрый взгляд на код ассемблера Go**

Даже для чего-то такого простого, как печать строки, runtime Go обрабатывает удивительное количество работы. Функция print переводится в несколько вызовов внутри пакета runtime. Компилятор сначала поднимает все вычисления аргументов перед получением блокировки. Затем, основываясь на типе аргумента, он выбирает соответствующую функцию runtime:

- Для булевых значений: runtime.printbool
- Для целых чисел: runtime.printint
- Для строк: runtime.printstring
- Для чисел с плавающей точкой: runtime.printfloat
- Для slice'ов: runtime.printslice
- И другие, найденные в src/runtime/print.go.

Каждая из этих функций работает между runtime.printlock() и runtime.printunlock() для обеспечения потокобезопасности, предотвращая перекрытие вывода, когда несколько goroutine'ов работают одновременно.

В конце runtime.morestack_noctxt вмешивается для автоматической обработки роста стека. Этот механизм позволяет Go настраивать размер стека goroutine на лету — важная функция, которую мы рассмотрим более подробно позже.
