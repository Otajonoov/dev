# 11. Garbage Collector

Bu bosqichda haqiqiy Go GC'ni almashtirmaymiz. O'z allocator va type descriptor modelingiz ustida toy GC yozamiz.

## Chegara

Go dasturining real heapini Go runtime GC boshqaradi. Siz yozadigan GC faqat sizning lab heap/arena ichidagi objectlarni boshqaradi.

```text
Go heap          -> Go runtime GC
runtime-lab heap -> siz yozgan toy GC
```

## Bosqich 1: Manual Roots

Avval rootlarni qo'lda berasiz:

```go
type GC struct {
    heap  *Heap
    roots []Handle
}

func (g *GC) AddRoot(h Handle)
func (g *GC) RemoveRoot(h Handle)
func (g *GC) Collect()
```

## Object Header

Har heap object headerga ega bo'lsin:

```go
type ObjHeader struct {
    size   uintptr
    typ    *TypeDesc
    marked bool
    next   Handle
}
```

Pointerlarni off-heap ichida Go pointer sifatida emas, `Handle`/offset sifatida saqlang.

## Type Pointer Mask

GC qaysi field reference ekanini bilishi kerak:

```go
type TypeDesc struct {
    Name       string
    Size       uintptr
    Align      uintptr
    PtrOffsets []uintptr
}
```

Mark paytida:

```text
object -> TypeDesc -> PtrOffsets -> child handles
```

## Bosqich 2: Mark-Sweep

Algorithm:

```text
Mark:
  rootlardan boshlash
  stack/queue bilan reachable objectlarni belgilash

Sweep:
  heapdagi objectlarni yurish
  marked bo'lmaganlarni free listga qaytarish
  marked bo'lganlarni keyingi cycle uchun unmark qilish
```

## Bosqich 3: Tri-Color Model

Ranglar:

- white: hali ko'rilmagan yoki unreachable
- gray: topilgan, lekin childlari scan qilinmagan
- black: object va childlari scan qilingan

Invariant:

```text
black object white objectga write barrier'siz pointer tutmasligi kerak
```

## Bosqich 4: Write Barrier Simulator

Concurrent GC'ni tushunish uchun write barrier yozing:

```go
func (g *GC) WritePointer(obj Handle, field uintptr, target Handle)
```

Barrier vazifasi:

- assignmentni bajarish
- GC cycle active bo'lsa targetni gray qilish
- invariant buzilmasligini tekshirish

## Bosqich 5: Incremental/Concurrent GC Simulator

Real concurrent GC emas, scheduler tick bilan incremental qiling:

```go
func (g *GC) Step(budget int)
```

Har step ma'lum miqdorda gray object scan qiladi.

## Mark Assist

Allocator GC active paytida yordam berishi mumkin:

```text
allocation ko'paysa -> mutator mark ishining bir qismini qiladi
```

Lab modeli:

```go
func (g *GC) Assist(bytesAllocated uintptr)
```

## Testlar

- unreachable object sweep qilinadi
- root object saqlanadi
- graph traversal cycle bilan ishlaydi
- pointer mask to'g'ri scan qiladi
- write barrier invariantni saqlaydi
- incremental step oxirida full mark bilan bir xil natija beradi
