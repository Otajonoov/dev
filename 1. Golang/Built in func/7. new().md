```go
func newobject(typ *_type) unsafe.Pointer {
    return mallocgc(typ.Size_, typ, true)
}
```

```txt
1. **Compiler tomonidan maxsus ishlov** 
    - `new(T)` yozganingizda, compiler buni `runtime.newobject` ga aylantiradi 
	- Bu builtin funksiya, oddiy funksiyalardan farq qiladi 
	- Compiler type ma'lumotlarini (`*_type`) yaratadi va `newobject`ga uzatadi 
	  
2. Runtime call chain:
   
    new(T) 
     ↓ 
     compiler transforms
      ↓
      runtime.newobject(typ)
       ↓ 
       runtime.mallocgc(typ.Size_, typ, true)
```

### `mallocgc` hotira ajratish strategiyasi

Go runtime 3 xil hotira ajratish usulini qo'llaydi:

#### **1. Tiny allocation (< 16 bytes, pointer yo'q)**
```
┌─────────────────────────────────┐
│  Tiny Allocator                 │
│  ┌──────┬──────┬──────┬──────┐  │
│  │ 8B   │ 4B   │ 2B   │ 2B   │  │ → Birgalikda paketlanadi
│  └──────┴──────┴──────┴──────┘  │
│  Total: 16 bytes                │
└─────────────────────────────────┘
```

- Kichik (16 byte dan kam), pointer'siz obyektlar uchun
- Bir nechta kichik allocation'lar bitta 16-byte blockka paketlanadi
- Memory overhead'ni kamaytiradi
- Misollar: `bool`, `uint8`, kichik struct'lar

#### **2. Small allocation (16 bytes - 32KB)**
```
┌─────────────────────────────────────┐
│  mcache (thread-local)              │
│  ┌──────────────────────────────┐   │
│  │ Size Class 1: 8 bytes        │   │
│  │ Size Class 2: 16 bytes       │   │
│  │ Size Class 3: 24 bytes       │   │
│  │ ...                          │   │
│  │ Size Class N: 32768 bytes    │   │
│  └──────────────────────────────┘   │
└─────────────────────────────────────┘
```

- 67 xil oldindan belgilangan **size class** mavjud
- Har bir size class uchun mcache'da tayyor span'lar
- Lock'siz allocation (thread-local cache)
- Internal fragmentation mavjud (masalan, 25 byte kerak bo'lsa, 32 byte ajratiladi)

**Size class jadvali (masalan):**
```
Size Class | Object Size | Objects per Span
-----------|-------------|------------------
1          | 8 bytes     | 512
2          | 16 bytes    | 256  
3          | 24 bytes    | 170
...
67         | 32768 bytes | 1
```

#### **3. Large allocation (> 32KB)**
```
┌─────────────────────────────────────┐
│  mheap (global heap)                │
│  ┌──────────────────────────────┐   │
│  │  Span 1: 64KB                │   │
│  │  Span 2: 128KB               │   │
│  │  Span 3: 256KB               │   │
│  └──────────────────────────────┘   │
└─────────────────────────────────────┘
```

- To'g'ridan-to'g'ri mheap'dan ajratiladi
- Mutex lock kerak (global resource)
- Overhead kamroq, lekin sekinroq (lock tufayli)

## Qaysi type uchun qaysi mexanizm?

### Type kategoriyalari
```go
// 1. TINY ALLOCATION (< 16 bytes, no pointers)
type TinyStruct struct {
    a uint8  // 1 byte
    b bool   // 1 byte
}
// Size: 2 bytes → Tiny allocator

// 2. SMALL ALLOCATION (16B - 32KB)
type SmallStruct struct {
    a int64    // 8 bytes
    b *string  // 8 bytes (pointer)
    c [100]byte // 100 bytes
}
// Size: 116 bytes → Small allocator (128 byte size class)

// 3. LARGE ALLOCATION (> 32KB)
type LargeStruct struct {
    data [40000]byte
}
// Size: 40000 bytes → Large 
```

### Pointer mavjudligi muhim

```go
// Pointer'siz (noscan) - GC scan qilmaydi
type NoPointers struct {
    x int

// Pointer'li (scan kerak) - GC scan qiladi
```
