### Сравнимость структур и семантика равенства

Go предоставляет чёткий набор типов, которые **по умолчанию сравнимы**: integer, float, string, pointer, channel, array и struct. 
Slice, map и function **не сравнимы**.

Однако, если struct или array содержит любой из этих несравнимых типов, он становится несравнимым и не может использоваться с `==` или `!=`.

Для того чтобы struct был сравнимым, **все его поля** должны быть сравнимыми:

```go
type Person struct {
    Emails []string
}

func main() {
    a := Person{}
    b := Person{}
    fmt.Println(a == b)
}
// compile error: invalid operation: a == b (struct containing []string cannot be compared)
```

Даже если две структуры выглядят одинаково, их не всегда можно сравнить. 
Это особенно верно для именованных структур с идентичными полями, но **разными именами типов**:

```go
type Person struct {
    Name string
}

type Employee struct {
    Name string
}

func main() {
    p1 := Person{Name: "John"}
    p2 := Person{Name: "Doe"}
    e := Employee{Name: "John"}

    fmt.Println(p1 == p2)  // Output: false
    fmt.Println(p1 == e)   // invalid operation: p1 == e (mismatched types Person and Employee)
}
```

Хотя `Person` и `Employee` имеют одно и то же поле одного типа, Go трактует их как **совершенно разные типы**. Имена типов значимы. 
Наличие идентичных полей недостаточно, если определения типов названы по-разному.
Это контрастирует с анонимными структурами, где сравнение основано исключительно на структуре.

Анонимные структуры могут сравниваться с именованными, если их поля точно совпадают по типу, имени и порядку:

```go
type Person struct {
    Name string
}

type Employee struct {
    Name string
}

func main() {
    p1 := Person{Name: "John"}
    p2 := Employee{Name: "Doe"}
    e := struct{ Name string }{Name: "John"}

    fmt.Println(p1 == e)   // Output: true
    fmt.Println(p2 == e)   // Output: false
    fmt.Println(p1 == p2)  // invalid operation: p1 == p2 (mismatched types Person and Employee)
}
```

Это поведение может показаться неинтуитивным. 
Анонимная структура `e` может быть сравнена и с `Person`, и с `Employee`, если поля точно совпадают. Однако именованные структуры `Person` и `Employee` не могут сравниваться напрямую, так как Go применяет строгие границы типов на основе имён.

Вот интересный вопрос: если две структуры сравнимы, можно ли присвоить одну другой? Полезно остановиться и подумать.

Согласно спецификации Go:

> _«В любом сравнении первый операнд должен быть assignable к типу второго операнда, или наоборот.»_ — The Go Programming Language Specification

```go
a := struct{ Name string }{Name: "John"}
b := struct{ Name string }{Name: "Doe"}
a = b // Это работает без проблем
```

Это присваивание работает, потому что структура двух значений идентична. 
При сравнении или присваивании безымянных структур компилятор проверяет, что имена полей, типы, порядок и tag все согласованы.

**Assignability** играет ключевую роль здесь, но не покрывает всё. 
Tag **учитываются** при сравнении, но **игнорируются** при присваивании:

```go
func (x *operand) assignableTo(check *Checker, T Type, cause *string) (bool, Code) {
    ...
    V := x.typ
    // Underlying types V и T идентичны,
    // хотя бы один из V или T не является именованным типом,
    // и ни V, ни T не являются type parameter.
    if Identical(Vu, Tu) && (!hasName(V) || !hasName(T)) && Vp == nil && Tp == nil {
        return true, 0
    }
    ...
}

func (c *comparer) identical(x, y Type, p *ifacePair) bool {
    ...
    case *Struct:
        if y, ok := y.(*Struct); ok {
            if x.NumFields() == y.NumFields() {
                for i, f := range x.fields {
                    g := y.fields[i]
                    if f.embedded != g.embedded ||
                        !c.ignoreTags && x.Tag(i) != y.Tag(i) ||
                        !f.sameId(g.pkg, g.name) ||
                        !c.identical(f.typ, g.typ, p) {
                        return false
                    }
                }
                return true
            }
        }
        return false
}
// _Figure 58. assignableTo (src/cmd/compile/internal/types2/operand.go)_
```


Метод `identical` в компиляторе показывает, как принимается это решение. Флаг `c.ignoreTags` определяет, включаются ли tag в сравнение:

- Когда установлен в `false` — tag проверяются при сравнении или проверке идентичности типов.
- Когда установлен в `true` — tag игнорируются, что происходит при conversion или assignment.

В результате Go позволяет assignment и type conversion между структурами с совпадающими именами полей и типами, даже если их **tag отличаются**:

```go
func main() {
    p := Person{Name: "John"}
    a := struct {
        Name string `json:"hello"`
    }{Name: "John"}

    fmt.Println(Person(a)) // Output: {John }
    fmt.Println(a == p)    // invalid operation: a == p (mismatched types ...)
}
```

Это может показаться тонкой технической деталью, но имеет практическую пользу. Игнорирование tag при conversion позволяет большую гибкость при работе с совместимыми структурами из разных источников. Например, вы можете получать данные из внешних систем, использующих разные struct tag для сериализации, в то время как основные определения полей остаются теми же.
