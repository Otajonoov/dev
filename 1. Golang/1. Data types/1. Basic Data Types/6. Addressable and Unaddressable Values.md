Не каждое значение в Go имеет адрес памяти. Addressable (адресуемые) значения — это те, которые привязаны к определенному местоположению в памяти — это те, с которыми мы работали до сих пор:

- Переменные, объявленные на уровне пакета или функции.
- Поля внутри struct (пока сам struct адресуемый).
- Элементы массива, который имеет адрес.
- Элементы в slice'ах, независимо от того, адресуем ли сам slice или нет.

В отличие от этого, unaddressable (неадресуемые) значения не могут быть ссылаться с использованием оператора &. Эти значения часто появляются, когда Go создает временные результаты, такие как возвращаемое значение функции, которые не привязаны к стабильному адресу памяти.
```go
func returnArray() [3]int {
  return [3]int{1, 2, 3}
}

func returnStruct() Person {
  return Person{Name: "John", Age: 20}
}

func main() {
  // Ошибка: cannot take the address of returnArray() (value of type [3]int)
  array := &returnArray()
  
  // Ошибка: cannot take the address of returnArray()[0] (value of type int)
  arrayElement := &returnArray()[0]
  
  // Ошибка: cannot take the address of returnStruct() (value of type Person)
  person := &returnStruct()
  
  // Ошибка: cannot take the address of returnStruct().Name (value of type string)
  personName := &returnStruct().Name
}
```


Возвращаемое значение функции неадресуемо, потому что оно не имеет стабильного местоположения памяти, на которое вы можете надежно ссылаться. Хотя значение существует в регистрах или памяти в момент возвращения, это хранилище временное. Малые возвращаемые значения обычно хранятся в регистрах, в то время как большие записываются напрямую в stack frame вызывающей стороны.

Другие значения также могут быть неадресуемыми, даже вне возвращаемых значений функций:
```go
func main() {
  const constant = 10
  // Ошибка: cannot take address of constant (untyped int constant 10)
  constantAddress := &constant
  
  // Ошибка: cannot take address of 10 (untyped int constant)
  literalAddress := &10
  
  // Ошибка: cannot take address of (10 + 20) (untyped int constant 30)
  expressionAddress := &(10 + 20)
  
  aMap := map[int]string{1: "one"}
  // Ошибка: cannot take address of aMap[1] (map index expression of type string)
  mapElementAddress := &aMap[1]
}
```

Константы и литералы неадресуемы, и компилятор Go применяет это правило, даже если они могут быть сохранены в секции памяти только для чтения. В некоторых случаях константы и литералы встраиваются напрямую в поток инструкций, что означает, что у них вообще нет адреса памяти.

При работе с pointer'ами вы храните адрес памяти переменной. Каждый pointer имеет конкретный тип, что означает, что он может указывать только на переменные этого типа. Нулевое значение pointer'а — nil, и разыменование nil pointer'а вызовет runtime panic. Go обеспечивает безопасность памяти, запрещая арифметику pointer'ов — если только вы не работаете с пакетом unsafe.
