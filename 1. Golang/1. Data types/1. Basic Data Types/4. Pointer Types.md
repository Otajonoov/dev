Каждая переменная в Go имеет место в памяти, и это место имеет адрес. Pointer — это переменная, которая хранит этот адрес вместо того, чтобы хранить значение напрямую. Хотя pointer технически мог бы хранить любой адрес в памяти, на практике он в основном используется для ссылки на адрес переменной.
```go
func main() {
  var a int = 10
  var b *int         // Объявление pointer'а на int
  b = &a             // Присвоение b адреса a
  
  println(b)         // Печатает адрес памяти a, например, 0x14000050738
  println(*b)        // Печатает 10, значение, хранящееся по адресу памяти, на который указывает b
  
  *b = 20            // Обновляет значение по этому адресу (a становится 20)
  println(a)         // Печатает 20
  println(&b)        // Печатает собственный адрес памяти b, например, 0x14000050748
}
```

Вот что происходит шаг за шагом:

- **Объявление pointer'а**: Используйте * перед типом (*int, *float64, *[]int), чтобы объявить переменную, которая хранит адрес памяти этого конкретного типа.
- **Получение адреса памяти переменной**: Используйте & перед именем переменной (&a), чтобы получить ее адрес в памяти.
- **Разыменование pointer'а**: Используйте * перед pointer'ом (*b), чтобы получить доступ к значению, хранящемуся по адресу памяти, на который он указывает.
- **Обновление значения по цели pointer'а**: Присвойте новое значение *b, чтобы обновить исходную переменную (a).
- Поскольку b также является переменной, вы можете получить ее собственный адрес памяти, используя &b.
- Хотя шестнадцатеричные адреса памяти могут выглядеть странно, помните, что это просто числа. Go по умолчанию отображает адреса памяти в шестнадцатеричном формате, потому что он более краткий и легче читается для больших значений. Если вы предпочитаете видеть адрес в десятичном формате, вы можете использовать fmt.Printf("%d", b), что может вывести что-то вроде 1374389864264. Это показывает, что pointer, за кулисами, — это просто число размером с адрес, в то время как Go применяет семантику ссылок для того, как используются pointer'ы.
- pointer представлен как беззнаковое целое число (uint). На 32-битной системе он занимает 4 байта; на 64-битной системе он занимает 8 байт.
- Другой способ создать pointer в Go — использовать встроенную функцию new().
- `new(int)` zero value bilan xotira ajratadi va pointer qaytaradi. Bu odatda primitive type'lar uchun ishlatiladi. Struct'lar uchun odatda `&Book{}` yoziladi.
- **nil pointer - xavfli!** nil pointer - bu hech qayerga ishora qilmaydigan pointer (0x0). Uni dereference qilishga urinish `panic` keltirib chiqaradi. 

new() как выделяет нулевое значение для данного типа, так и предоставляет pointer, указывающий на него:

```go
var a *int
var b *int = new(int)

a    // nil
b    // 0x14000110018
*b   // 0
```

- `var a *int`: Объявляет a как pointer на int, но не присваивает ему значение, поэтому по умолчанию он равен nil.
- `var b *int = new(int)`: Выделяет память для int, инициализированного в 0, и возвращает pointer на эту память.

Распространенное заблуждение, что new() всегда выделяет память на heap просто потому, что возвращает pointer. Хотя многие функции, возвращающие pointer'ы, действительно включают выделение на heap, компилятор Go достаточно умен, чтобы оптимизировать это.

Если pointer, созданный с new(), используется только внутри функции и не "убегает" в другие части программы, компилятор может выделить его на stack вместо этого. Это делает операцию быстрее и эффективнее, поскольку память stack автоматически очищается, когда функция завершается.

---

Pointer'ы в Go специфичны для типа, что означает, что они типобезопасны. Когда у вас есть pointer на значение типа T, он записывается как *T — где T — базовый тип, а *T — тип pointer'а.

Вы не можете смешивать и сопоставлять pointer'ы разных типов. Если попытаетесь, Go остановит вас во время компиляции:

go

```go
var a int = 10
var b *string = &a  // cannot use &a (type *int) as type *string in assignment
```

Адрес памяти — это просто число под капотом. Но Go требует точности в том, на какой тип данных ссылается ваш pointer — он не позволит вам обойти это правило.

По умолчанию нулевое значение pointer'а — nil, что означает, что он не указывает на какое-либо валидное местоположение в памяти (часто представляется как 0x0). Если вы знакомы с null из других языков, он работает так же в Go.

---
Вы могли слышать, как разработчики Go кричат об ошибках типа "nil pointer dereference" или "nil pointer panic". Это происходит, когда вы пытаетесь получить доступ к значению, хранящемуся в nil pointer, что вызывает ошибку времени выполнения — и да, это серьезная ошибка:
```go
func main() {
  var a *int
  println(*a) 
   // panic: runtime error: invalid memory address or nil pointer dereference
}
```

Поскольку nil pointer не ссылается на какую-либо валидную память, Go не может получить из него значение. Вместо того чтобы позволить программе продолжить и рисковать непредсказуемым поведением, Go немедленно останавливает выполнение и вызывает panic, чтобы предотвратить дальнейший ущерб.

**Qiziq faktlar:**

- Pointer ichkarida oddiy uint (32-bit'da 4 byte, 64-bit'da 8 byte)
- Hexadecimal ko'rinishi faqat qulaylik uchun
- Stack frame'da qo'shni o'zgaruvchilar odatda yaqin address'larga ega
