Go предоставляет два типа с плавающей точкой: float32 и float64. Они соответствуют числам с плавающей точкой одинарной и двойной точности:
```go
var a float32 = 0.1
var b float64 = 0.0001
```

Несмотря на использование того же количества битов, что и целые числа (int32, int64), числа с плавающей точкой могут представлять гораздо больший диапазон значений за счет точности:
```go
float32: 1.4e-45 до 3.4e+38
float64: 4.9e-324 до 1.8e+308
```

---
Когда тип с плавающей точкой не определен явно, по умолчанию всегда используется float64, независимо от архитектуры:
```go
a := 0.1
fmt.Printf("Type of a is %T\n", a)  // "Type of a is float64"
```

Тип по умолчанию float64 выбран для точности. float64 — стандарт в библиотеке Go, потому что он обеспечивает большую точность для вычислений с плавающей точкой. В отличие от этого, float32 теряет точность быстро.

---

Числа с плавающей точкой, или литералы, могут быть записаны в трех основных форматах: десятичный, научная нотация и шестнадцатеричный.

- **Десятичная форма**: Стандартный формат с точкой, разделяющей целую и дробную части.
- **Научная нотация**: Полезна для очень больших или очень малых чисел, записывается как базовое число, за которым следует e или E и экспонента.
- **Шестнадцатеричная форма**: Похожа на шестнадцатеричные целые числа, но используется для чисел с плавающей точкой, с основанием 2 вместо 10.
```go
// Десятичная форма
d1 := 0.1
d2 := 1.
d3 := .1

// Научная нотация
s1 := 1.2e3   // 1.2 * 10^3 = 1200
s2 := 1.E-3   // 1 * 10^-3 = 0.001
s3 := .2e3    // 0.2 * 10^3 = 200

// Шестнадцатеричная форма
h1 := 0x1.2p3 // (1 + 2/16) × 2^3 = 1.125 × 8 = 9
h2 := 0x1P3   // 1 * 2^3 = 8
h3 := 0X1.p3  // 1 * 2^3 = 8
```

---
**Специальные значения с плавающей точкой: +Inf, -Inf и NaN**

Деление с плавающей точкой на ноль работает иначе, чем деление целых чисел. Вместо вызова ошибки Go возвращает специальные значения в зависимости от ситуации:
```go
func main() {
  zero := 0.0
  println(1 / zero)      // +Inf
  println(-1 / zero)     // -Inf
  println(zero / zero)   // NaN
}
```
