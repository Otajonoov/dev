Iterator - bu ma'lumotlar ustidan aylanish uchun mexanizm/pattern, for esa - bu aylanish uchun ishlatiladigan sintaksis (loop konstruktsiyasi)

---
---

Iterator vs For
## Farqi:

**`for` - bu loop sintaksisi** (dasturlash tili konstruktsiyasi):
```go
for i := 0; i < 10; i++ {
    fmt.Println(i)
}
```


**Iterator - bu pattern/mexanizm** (ma'lumotlar ustidan aylanish usuli):
```go
// Bu iterator - funksiya
func Numbers(n int) func(yield func(int) bool) {
    return func(yield func(int) bool) {
        for i := 0; i < n; i++ {
            if !yield(i) {
                return
            }
        }
    }
}
```


## Aloqa:

`for` loop **iterator ustidan** ishlashi mumkin:
```go
// for loop iterator bilan ishlatyapti
for num := range Numbers(5) {
    fmt.Println(num)
}
```

## Analogy:

- **Iterator** = Muzey gidi (sizni yo'lga soladi, nima ko'rsatishni biladi)
- **`for` loop** = Siz muzey bo'ylab yurganingiz (harakat qilish jarayoni)

## Asosiy farq:

```go
// 1. Oddiy for loop - siz hamma narsani boshqarasiz
for i := 0; i < len(slice); i++ {
    fmt.Println(slice[i])
}

// 2. Iterator - iterator sizga elementlarni beradi
for item := range myIterator() {
    fmt.Println(item)
}
```

Iterator ichida `for` loop bo'lishi mumkin (ko'pincha ham shunday), lekin iterator o'zi `for` emas - u **abstraktsiya**, ya'ni "ma'lumotlarni birma-bir berish usuli".

Iterator vs range
## Farqi:

**`range` - bu keyword** (dasturlash tili kalit so'zi):
```go
for item := range something {
    // ...
}
```


**Iterator - bu funksiya** (ma'lum signature'ga ega):
```go
func Numbers(n int) func(yield func(int) bool) {
    return func(yield func(int) bool) {
        for i := 0; i < n; i++ {
            if !yield(i) { return }
        }
    }
}
```

## Aloqa:

`range` keyword **iterator ustidan** aylanishi mumkin:
```go
// range iterator funksiyani qabul qilyapti
for num := range Numbers(5) {
    fmt.Println(num)
}
```

## Go 1.23 gacha va undan keyin:

**Ilgari (Go 1.23 gacha):**
```go
// range faqat built-in turlar bilan ishlardi
for i, v := range slice { }    // ✓
for k, v := range map { }       // ✓
for ch := range channel { }     // ✓
for i, r := range string { }    // ✓
```

**Hozir (Go 1.23+):**
```go
// range endi iterator funksiyalar bilan ham ishlaydi
for item := range MyIterator() { }  // ✓

## Analogy:
// **Iterator** = Musiqa pleylisti (qo'shiqlarni tartib bilan berish mexanizmi)
// **`range`** = "Play" tugmasi (pleylisti ijro etish buyrug'i)

## Xulosa:
range + iterator = birgalikda ishlaydi
range ≠ iterator
iterator ≠ range

range - bu qanday aylanish (sintaksis)
iterator - bu nima ustidan aylanish (ma'lumot manbai)
```
