Модель MPG — это сердце того, как Go управляет конкурентностью. Она состоит из трёх основных частей:

- **M (Machine)** — это OS thread, который выполняет ваш код. Этот поток также может заблокироваться, если ему нужно что-то подождать от операционной системы.
- **P (Processor)** — это не аппаратное ядро CPU. Вместо этого это то, чем управляет Go runtime. Он даёт потоку всё необходимое для запуска goroutine.
- **G (Goroutine)** — это лёгковесная функция, выполняющаяся параллельно с другими goroutine.

Machine (OS thread) не может запустить goroutine самостоятельно. Ему нужен processor. Processor даёт потоку правильный контекст для запуска goroutine. Можно представить это так: поток несёт processor, а processor несёт goroutine. Если потоку нужно выполнить блокирующую операцию, например system call, он отдаёт свой processor. Это позволяет другому потоку захватить processor и продолжить выполнение других goroutine.

Количество потоков и goroutine может меняться во время работы программы. Количество processor, однако, остаётся фиксированным и может быть задано вручную через `runtime.GOMAXPROCS(n)`. Если у потока нет processor, он просто ждёт. Он не может выполнять Go-код без него.
