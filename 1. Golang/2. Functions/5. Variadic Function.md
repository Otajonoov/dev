
Функция становится «variadic»,когда её последний параметр использует модификатор `...`
Это указывает компилятору собирать любое количество переданных значений (ноль, одно или несколько) в один slice.

Внутри функции вы работаете с этим параметром точно так же, как с любым другим slice:

```go
func sum(nums ...int) int {
    total := 0
    for _, v := range nums {
        total += v
    }
    return total
}
sum(1, 2, 3, 4, 5)
sum(1, 2)
```

Вариадические функции широко используются в стандартной библиотеке, например `fmt.Println` или встроенная функция `append`:

```go
func Println(a ...any) (n int, err error) {}
func append(slice []Type, elems ...Type) []Type {}
```

Если у вас уже есть slice и вы хотите передать его в вариадическую функцию, вам нужно использовать завершающий `...` для «explode» slice обратно в отдельные аргументы:

```go
var s = []int{1, 2, 3, 4, 5}
sum(s...)
```

При вызове вариадической функции:

- Компилятор упаковывает дополнительные аргументы в новый slice и передаёт его функции.
- Если вы вызываете функцию со slice и завершающим `...`, компилятор передаёт slice напрямую в функцию.

Синтаксис `func(slice...)` не «распаковывает» элементы slice — он передаёт сам slice, поэтому изменения элементов внутри функции будут видны снаружи:

```go
func sum(nums ...int) int {
    total := 0
    for _, v := range nums {
        total += v
    }
    nums[0] = 100
    return total
}
func main() {
    s := []int{1, 2, 3, 4, 5}
    fmt.Println(sum(s...)) // 15
    fmt.Println(s)         // [100 2 3 4 5]
}
```

Когда аргументы не передаются, создаётся `nil` slice, а не пустой slice.

### Функция init

Функция `init` — одна из двух специальных функций в Go (другая — `main`). 
Вы можете иметь более одной функции `init` в пакете или даже в одном файле:

```go
package main

func init() {
    println("Init 1 called")
}
func init() {
    println("Init 2 called")
}
func main() {
    println("Main called")
}
// Init 1 called
// Init 2 called
// Main called
```

В этом примере есть две функции `init()`, и обе вызываются до `main()`. 
Процесс сбора этих функций `init()` и определения порядка их выполнения обрабатывается во время компиляции.

Каждый пакет в Go может зависеть от других пакетов. 
Пакет инициализируется только после того, как все его зависимости инициализированы. 
Также перед вызовом любой функции `init()` все переменные уровня пакета должны быть настроены.

Поэтому, если вы объявляете переменную уровня пакета с начальным значением, эта переменная устанавливается до запуска любой функции, включая функции `init()` в этом пакете:

```go
var x int = 10
func init() {
    println(x)
}
// 10
```

После настройки всех глобальных переменных запускаются функции `init()`. 
Основная цель `init()` — выполнить любую настройку или подготовку, необходимую перед использованием пакета. 
Если у пакета более одной функции `init()`, они выполняются в порядке их появления в исходном коде.
