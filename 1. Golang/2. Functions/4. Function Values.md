
Когда Go компилирует функции, он создаёт несколько типов символов. 
Для каждой функции существует не только код самой функции, но и связанные метаданные и специальные символы значений функций.

Символ значения функции особенно важен. 
Он позволяет Go поддерживать функции первого класса. 
Это означает, что вы можете передавать функции как значения, хранить их в переменных и вызывать косвенно.

Этот символ следует определённому правилу именования:

```go
func FuncSymName(s *types.Sym) string {
    return s.Name + "·f"
}
```

если у вас есть функция `main.foo`, компилятор создаёт символ с именем `main.foo·f`. 
Символ значения функции — это не фактический код функции. 
Это структура данных, которая содержит указатель на реальный код функции.

Когда вы присваиваете функцию переменной:

```go
func add(x, y int) int {
    return x + y
}
f := add
f(1, 2)
```

Компилятор Go создаёт символ значения функции (`add·f`) и настраивает его в вашем бинарном файле:

```go
type funcval struct {
    fn uintptr
    // variable-size, fn-specific data here
}
```

Символ значения функции фактически является структурой `funcval`. 
Эта структура хранит указатель на код функции. 
Если функция является closure и захватывает переменные из своей среды, то `funcval`также будет хранить эти захваченные переменные сразу после поля `fn`.

Чтобы увидеть адрес символа `add·f` в runtime, вы можете использовать функцию `println`

```go
//go:noinline
func add(x, y int) int {
    return x + y
}
func main() {
    f := add
    println(f)
}
// Output:
// 0x100bce7f0
```

Здесь `f` — это указатель на символ значения функции `add·f`. Иными словами, `f` имеет тип `*funcval`.

![](../../assets/obsidian-images/Pasted%20image%2020260218143011.png)

При вызове функции-значения `f(1, 2)` компилятор следует соглашению Go о вызове:

- `MOVD main.f-8(SP), R26` загружает указатель `funcval` из локальной переменной `f` в стеке в R26. На ARM64 R26 служит регистром контекста closure. Этот шаг требуется ABI Go для всех вызовов функций-значений.
- `(R26), R2`: поскольку R26 указывает на структуру `funcval`, а первое поле в `funcval` — это `fn` (адрес функции), эта инструкция загружает адрес фактической точки входа функции в регистр R2.
- `$1, R0` и `$2, R1`: загружают аргументы функции.
- `CALL (R2)`: вызывает функцию, адрес которой был загружен из структуры `funcval`.

![](../../assets/obsidian-images/Pasted%20image%2020260218143027.png)

Процесс в простом псевдокоде:

```go
var f = &main.add·f
R26 = f       // MOVD main.f-8(SP), R26
R2 = *R26     // MOVD (R26), R2
R0 = 1
R1 = 2
CALL (R2)     // (*R2)(R0, R1) или f(1, 2)
```

### Как работают closure

Существует важное различие между анонимной функцией, которая не захватывает никаких переменных из окружающей среды (это называется **trivial closure**), и той, которая их захватывает (**non-trivial closure**).

Non-trivial closure не имеет своего символа значения функции в секции RODATA. 
Вместо этого её символ значения функции создаётся динамически в runtime вместе с захваченными переменными. Пример non-trivial closure:

```go
func main() {
    x := 1
    f := func(y int) int {
        return x + y
    }
    f(2)
    println(&f)
}
```

В ассемблерном коде нет символа значения функции вроде `main.func1·f`, а структура `funcval` создаётся динамически:

1. `MOVD $main.main.func1(SB), R1` — помещает адрес функции непосредственно в первое поле структуры `funcval`.
2. `MOVD $1, R1` — загружает значение 1 (значение `x`) и сохраняет его в следующей позиции структуры closure. Именно так closure «захватывает» переменную — она копирует значение в closure в момент её создания.

Весь процесс создания non-trivial closure можно резюмировать следующим псевдокодом:

```go
func main() {
    funcval := funcval{
        fn:   $main.func1,
        arg1: 1,
    }
    f := &funcval
    f(2)
}
```

Теперь посмотрим, что произойдёт, если изменить переменную `x` перед вызовом closure:

```go
func main() {
    x := 1
    f := func(y int) int {
        return x + y
    }
    x = 100
    f(2)
    println(&f)
}
```

В этом случае `x` захватывается по ссылке. 
Вместо хранения значения `$1` сохраняется адрес `x` в структуре closure. 
Функция closure затем разыменовывает этот адрес каждый раз, когда ей нужно получить значение.

Есть три условия, при которых компилятор Go захватывает по значению, а не по ссылке:

- Адрес переменной никогда не берётся.
- Переменная никогда не переприсваивается после захвата closure (включая внутри самой closure). Если она переприсваивается, она будет захвачена по ссылке.
- Размер переменной не превышает 128 байт.

Чтобы завершить этот раздел, рассмотрим более продвинутый пример:

```go
func returnAdder() func(int) int {
    x := 1
    return func(y int) int {
        x += y
        return x
    }
}
func main() {
    adder := returnAdder()
    println(adder(1)) // 2
    println(adder(2)) // 4
    println(adder(3)) // 7
}
```

Переменная `x` захватывается по ссылке, поскольку она изменяется внутри closure x += y Чтобы память `x` «оставалась живой», компилятор выделяет память в heap для `x`, которая переживёт stack frame функции. 
Ещё одно выделение в heap происходит для самой структуры closure:

```go
struct { 
	F uintptr
	X0 *int 
}
```

В функции `main` каждый вызов closure работает с одной и той же переменной `x`, выделенной в heap, через указатель, хранящийся в структуре closure. 
Вот почему значение накапливается: 
- первый вызов возвращает 2 (1+1)
- второй — 4 (2+2)
- третий — 7 (4+3).
