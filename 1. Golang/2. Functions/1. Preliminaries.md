### Stack Pointer, Program Counter и Argument Pointer


##### Stack frame

Stack frame — defer statement va panic recovery mexanizmi qanday ishlashining asosi. Profiling va tracing'ni tushunish uchun ham muhim.

Каждый раз, когда ваша программа вызывает функцию, processor выделяет блок памяти в стеке специально для этого вызова. Этот блок называется stack frame. Он содержит всё, что нужно функции для работы и для передачи управления обратно вызывающей функции после завершения.

> **Stack frame** — funksiya chaqiruvi uchun ajratilgan memory bloki. Odatda quyidagilarni saqlaydi: return address (funksiya tugagach CPU qaerga qaytishni bilish uchun), funksiyaga uzatilgan argumentlar, lokal o'zgaruvchilar uchun joy.

![](../../assets/obsidian-images/Pasted%20image%2020260218100902.png)

Memory'da stack **pastga** o'sadi — yuqori memory adreslardan past adreslarga. Stack'ga data push qilganingizda, stack pointer **kichiklashadi**. Funksiya tugaganida, uning stack frame'i olib tashlanadi, stack pointer oldingi joyiga qaytariladi va dastur to'xtatilgan joydan davom etadi.

Данный раздел сосредоточен на трёх важных понятиях: stack pointer (SP), program counter (PC) и управление аргументами функции.

---

##### Stack Pointer (SP)

**Stack Pointer (SP)** — har doim stack'ning **cho'qqisiga** (eng past memory adresi) ko'rsatadigan register. Har bir funksiya chaqirilganida SP pastga siljiydi — yangi stack frame uchun joy ochadi.

Каждая функция обычно получает собственный stack frame. 
В данном контексте «вершина» означает наименьший адрес памяти в стеке.

![](../../assets/obsidian-images/Pasted%20image%2020260218100939.png)

Каждый раз при вызове функции stack pointer движется вниз, чтобы освободить место для stack frame этой функции.

![](../../assets/obsidian-images/Pasted%20image%2020260218100955.png)

#### Argument Pointer (argp)

Когда вы вызываете функцию, вызывающая функция должна передать данные вызываемой функции. Это аргументы вашего вызова функции. 
Итак, куда именно в памяти попадают эти данные и как вызываемая функция их находит?

В Go аргументы функции могут передаваться двумя способами. 
Некоторые аргументы передаются через регистры processor.
Регистры — это очень быстрые места хранения внутри CPU. 
Но регистров мало. Если аргументов больше, чем регистров, или если аргумент слишком большой, то лишние аргументы помещаются в память, как правило в стек. 

Вызываемой функции (callee) нужен способ найти эти аргументы в памяти. 
Argument pointer предоставляет точку отсчёта. 
Он точно указывает callee, где начинаются его аргументы.

![](../../assets/obsidian-images/Pasted%20image%2020260218101043.png)

В зависимости от аппаратного обеспечения расположение может меняться. 
Однако argument pointer (argp) обычно указывает на тот же адрес, что и stack pointer, или чуть выше него. Обратите внимание, что argument pointer — это внутренняя концепция runtime/компилятора, а не переносимая архитектурная конструкция.

#### Program Counter (PC)

Program counter(PC),иногда называемый instruction pointer, это специальный регистр в CPU. 
Он отслеживает адрес памяти следующей инструкции, которую выполнит ваша программа. По мере выполнения кода CPU извлекает инструкции из памяти, декодирует их и выполняет одну за другой.

После выполнения каждой инструкции program counter обновляется и указывает на адрес следующей инструкции.

![](../../assets/obsidian-images/Pasted%20image%2020260218101641.png)

> **ПРИМЕЧАНИЕ** Эта диаграмма — упрощённый способ представить происходящее, и она не является технически точной. Program counter работает на уровне инструкций, а не на уровне строк кода. Одна строка кода может превратиться в несколько инструкций. Тем не менее диаграмма помогает показать, что делает program counter по мере выполнения инструкций.

Когда функция вызывает другую функцию, PC содержит адрес следующей инструкции в текущей функции. Перед переходом к новой функции этот адрес сохраняется как адрес возврата. Таким образом, когда вызванная функция завершается, она знает, куда вернуться и продолжить выполнение. Метод сохранения адреса возврата зависит от типа CPU:

- На **AMD64** инструкция `CALL` автоматически помещает адрес возврата (адрес сразу после вызова) в стек.
- На **ARM64** инструкция `BL` (Branch with Link, эквивалент `CALL` в AMD64) сохраняет адрес возврата в специальном link register (R30). Позже это значение сохраняется в stack frame вызванной функции.

Это различие важно для понимания того, как stack frame и вызовы функций работают на низком уровне.
