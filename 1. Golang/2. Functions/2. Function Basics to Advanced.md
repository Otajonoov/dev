
Функция — это самостоятельная последовательность операторов, выполняющая определённую задачу, которую можно вызывать из других частей программы.

В Go обычная функция начинается с ключевого слова `func`, за которым следуют имя функции, список параметров в скобках, информация о возвращаемых типах и тело в фигурных скобках. Например:

```go
func add(a int, b int) int {
    return a + b
}
println(add(1, 2)) // 3
```

Этот код объявляет функцию с именем `add`. Она принимает два целых числа и возвращает их сумму. Go всегда явно указывает возвращаемые типы. Если функция должна вернуть более одного значения, можно перечислить возвращаемые типы через запятую в одном наборе скобок:

```go
// функция, возвращающая частное и остаток
func divMod(numerator, denominator int) (int, int) {
    q := numerator / denominator
    r := numerator % denominator
    return q, r
}
q, r := divMod(17, 5) // q == 3, r == 2
```

На месте вызова вы можете захватить оба результата сразу, присвоив их переменным в одном операторе.

### Функции как объекты первого класса

#### Семантика передачи по значению

Функции в Go являются объектами первого класса. Это означает, что вы можете использовать функции как значения, так же как и любые другие типы данных. Вы можете присвоить функцию переменной, передать её в качестве аргумента другой функции, вернуть из функции или сохранить в структуре данных:

```go
// Объявление типа функции
var add func(int, int) int

// Функция как значение в map
var hammer32 = map[string]func(*uint32, int){
    ...
}

// Тип функции как параметр и возвращаемый тип
func OnceFunc(f func()) func() {
    ...
}
```

Значение по умолчанию для типа функции — `nil`. 
Функции относятся к категории «несравниваемых» типов, то есть вы не можете использовать операторы вроде `==` для сравнения функций (кроме сравнения с `nil`).

Тип функции в Go определяется её «сигнатурой». 
Когда мы говорим, что две функции имеют одинаковую сигнатуру, это означает, что типы их параметров и возвращаемых значений совпадают точно и в одном порядке:

```go
func add(a, b int) int {
    return a + b
}
func sub(c, d int) int {
    return c - d
}
func main() {
    f := add
    println(f(1, 2))
    f = sub
    println(f(1, 2))
}
```

Здесь `add(a, b int)` и `sub(c, d int)` имеют одинаковую сигнатуру. 
Имена параметров не имеют значения для сигнатуры функции.

Go не поддерживает перегрузку функций или перегрузку операторов. 
Нельзя определить несколько функций с одинаковым именем и разными параметрами. Также нельзя изменить смысл операторов для разных типов.

FAQ Go объясняет это так: «Диспетчеризация методов упрощается, если ей не нужно выполнять сопоставление типов. Опыт работы с другими языками показал нам, что наличие множества методов с одинаковыми именами, но разными сигнатурами иногда бывает полезным, но на практике может быть запутанным и ненадёжным. 
Сопоставление только по имени и требование согласованности типов было важным решением по упрощению системы типов Go.»

Когда речь идёт о сигнатуре функции, можно игнорировать имена параметров и возвращаемых значений. 
Это применяется как к определениям interface, так и к объявлениям функций:

```go
func add(int, int) int {
    return 0
}
type Add func(int, int) int
type Adder interface {
    Add(int, int) int
}
```

Go также поддерживает именованные возвращаемые значения, иногда называемые именованными результирующими параметрами. 
При использовании именованных возвращаемых значений вы можете дать каждому возвращаемому значению имя. Когда вы используете именованные возвращаемые значения, Go автоматически создаёт переменные с этими именами. 
Эти переменные начинают со своего нулевого значения сразу при начале функции:

```go
func calculateRectangle(dim1, dim2 float64) (area, perimeter float64) {
    area = dim1 * dim2
    perimeter = 2 * (dim1 + dim2)
    return
}
```

Хотя именованные возвращаемые значения могут иногда затруднять чтение кода при неосторожном использовании, они также могут улучшить документацию вашего кода. Они явно указывают в сигнатуре функции, что означает каждое возвращаемое значение.

Например, вот реальный пример из пакета `internal/singleflight` Go:

```go
func (g *Group) Do(key string, fn func() (any, error)) (v any, err error, shared bool) {}
```

Без именованных возвращаемых значений было бы непонятно, что представляет собой возвращаемое значение `bool`.

Иногда вам не нужны все возвращаемые значения функции.
Go позволяет игнорировать любые ненужные значения с помощью пустого идентификатора `_`
Вы можете использовать его столько раз, сколько нужно:

```go
area, _ := calculateRectangle(3, 4)
// Если функция возвращает более двух значений
area, _, _ := calculateRectangle(3, 4)
```

Go также предоставляет простой способ передавать множественные возвращаемые значения из одной функции непосредственно в другую функцию, при условии совпадения количества и типов.

Эта возможность позволяет вам связывать функции в цепочку без дополнительных переменных:

```go
func dimensions() (int, int, int) {
    return 5, 3, 2
}
func calculateVolume(length, width, height int) int {
    return length * width * height
}
calculateVolume(dimensions())
```

В этом примере `calculateVolume` может принимать результаты `dimensions` напрямую, потому что все они являются целыми числами и количества совпадают. Вы также можете написать функцию, которая просто возвращает результат другой функции, если их возвращаемые типы одинаковы:

```go
func dimensionWrapper() (int, int, int) {
    return dimensions()
}
```

Ещё одна важная деталь в Go заключается в том, что аргументы функции всегда передаются по значению. Это означает, что при передаче аргумента в функцию Go создаёт копию этого аргумента. Если вы работаете с «ссылочными типами», такими как slice, map или channel, то копируемое значение фактически является дескриптором. 
Этот дескриптор содержит указатель на базовые данные вместе с другими метаданными.

Эта идея может быть немного запутанной, если вы привыкли к таким языкам, как C# или Java. Возьмём следующий пример в Go:

```go
func modifySlice(s []int) {
    s = []int{1, 2, 3}
}
func main() {
    s := []int{4, 5, 6}
    modifySlice(s)
    fmt.Println(s) // [4 5 6]
}
```

В этом примере функция `modifySlice` присваивает `s` новый slice. 
Поскольку дескриптор slice передаётся по значению, это переприсваивание изменяет только локальную копию внутри функции. Оно не изменяет slice в вызывающей функции.

Вот почему исходный slice остаётся прежним. 
Это поведение отличается от того, как вещи работают в C#. Чтобы по-настоящему передать ссылку по ссылке в C#, вы используете ключевое слово `ref` или `out`:

```csharp
void ReplaceList(ref List<int> list) {
    list = new List<int> { 1, 2, 3 };
}
var myList = new List<int> { 4, 5, 6 };
ReplaceList(ref myList);
Console.WriteLine(string.Join(", ", myList)); // Output: 1, 2, 3
```

В этом примере C# сама ссылка передаётся по ссылке. 
Любое переприсваивание изменяет переменную и в вызывающем коде. 
Некоторые говорят, что в Go нет настоящей «передачи по ссылке». 
Это верно, в зависимости от того, имеете ли вы в виду передачу дескриптора или передачу настоящей ссылки, как в C#.

В Go, если вы изменяете элементы внутри slice вместо его переприсваивания, ваши изменения видны вызывающей функции:

```go
func modifySlice(s []int) {
    s[0] = 1
}
func main() {
    s := []int{4, 5, 6}
    modifySlice(s)
    fmt.Println(s) // [1 5 6]
}
```

Это работает потому, slice — это фактически небольшая struct. 
Она содержит указатель на массив, длину slice и его ёмкость. 
Когда вы передаёте slice в функцию, Go копирует эту struct — дескриптор или обёртку.

Итак, у вас получаются два разных дескриптора slice, оба указывающие на один и тот же массив в памяти. Если вы изменяете элементы slice, вы обновляете общий базовый массив, и изменения видны в обоих местах. 
Это происходит потому, что вы работаете через дополнительный уровень косвенности.
