
В Go нельзя определить именованную функцию внутри другой функции. 
Все именованные функции должны находиться на уровне пакета. 
Они не могут быть вложены. Однако, поскольку Go рассматривает функции как объекты первого класса, вы можете создавать анонимные функции (иногда называемые function literal) внутри других функций.

- Им не нужно имя
- их можно присваивать переменным
- передавать по ссылке
- возвращать из других функций

```go
func main() {
    x := "Hello, Go!"
    f := func() {
        fmt.Println(x)
    }
    f()
}
// Hello, Go!
```

Здесь мы определяем анонимную функцию с помощью `func() { ... }` и присваиваем её переменной `f`. Позже мы вызываем `f()` как любую другую функцию. 
Особенность анонимных функций в том, что они могут обращаться к переменным, находящимся в области видимости там, где функция создана. 
Они «замыкаются» на своей среде. Вот почему мы называем их closure.

Closure — это мощный инструмент.
Внутренняя функция может захватывать и сохранять доступ к переменным внешней функции даже после того, как внешняя функция вернула управление:

```go
func Incrementer() func() int {
    i := 0
    return func() int {
        i++
        return i
    }
}
func main() {
    inc := Incrementer()
    fmt.Println(inc()) // 1
    fmt.Println(inc()) // 2
    fmt.Println(inc()) // 3
}
```

В этом примере функция `Incrementer` возвращает другую функцию. 
Внутри `Incrementer` есть переменная `i`, которая начинается с 0.

Функция, которую возвращает `Incrementer`, является анонимной. 
Она прибавляет 1 к `i` при каждом вызове, затем возвращает новое значение `i`. 
Когда вы вызываете `Incrementer`, вы получаете обратно эту функцию вместе с её собственной переменной `i`, которая сохраняется между вызовами.

Go не просто копирует значения переменных из внешней функции в момент создания closure. Вместо этого он устанавливает длительную связь с этими переменными. 
Это позволяет closure читать и изменять их до тех пор, пока closure существует. 
Таким образом, closure и её среда разделяют состояние, которое сохраняется так же долго, как и сама closure.

### Обработка функций как данных: значения функций

Когда Go компилирует функции, он создаёт несколько типов символов. Для каждой функции существует не только код самой функции, но и связанные метаданные и специальные символы значений функций.

Символ значения функции особенно важен. Он позволяет Go поддерживать функции первого класса. Это означает, что вы можете передавать функции как значения, хранить их в переменных и вызывать косвенно.

Этот символ следует определённому правилу именования:

```go
func FuncSymName(s *types.Sym) string {
    return s.Name + "·f"
}
```

Итак, если у вас есть функция `main.foo`, компилятор создаёт символ с именем `main.foo·f`. Символ значения функции — это не фактический код функции. Это структура данных, которая содержит указатель на реальный код функции.

Когда вы присваиваете функцию переменной:

```go
func add(x, y int) int {
    return x + y
}
f := add
f(1, 2)
```

Компилятор Go создаёт символ значения функции (`add·f`) и настраивает его в вашем бинарном файле:

go

```go
type funcval struct {
    fn uintptr
    // variable-size, fn-specific data here
}
```

Символ значения функции фактически является структурой `funcval`. Эта структура хранит указатель на код функции. Если функция является closure и захватывает переменные из своей среды, то `funcval` также будет хранить эти захваченные переменные сразу после поля `fn`.

Чтобы увидеть адрес символа `add·f` в runtime, вы можете использовать функцию `println`:

go

```go
//go:noinline
func add(x, y int) int {
    return x + y
}
func main() {
    f := add
    println(f)
}
// Output:
// 0x100bce7f0
```

Здесь `f` — это указатель на символ значения функции `add·f`. Иными словами, `f` имеет тип `*funcval`.

При вызове функции-значения `f(1, 2)` компилятор следует соглашению Go о вызове:

- `MOVD main.f-8(SP), R26` загружает указатель `funcval` из локальной переменной `f` в стеке в R26. На ARM64 R26 служит регистром контекста closure. Этот шаг требуется ABI Go для всех вызовов функций-значений.
- `(R26), R2`: поскольку R26 указывает на структуру `funcval`, а первое поле в `funcval` — это `fn` (адрес функции), эта инструкция загружает адрес фактической точки входа функции в регистр R2.
- `$1, R0` и `$2, R1`: загружают аргументы функции.
- `CALL (R2)`: вызывает функцию, адрес которой был загружен из структуры `funcval`.

Процесс в простом псевдокоде:

go

```go
var f = &main.add·f
R26 = f       // MOVD main.f-8(SP), R26
R2 = *R26     // MOVD (R26), R2
R0 = 1
R1 = 2
CALL (R2)     // (*R2)(R0, R1) или f(1, 2)
```

### Как работают closure

Существует важное различие между анонимной функцией, которая не захватывает никаких переменных из окружающей среды (это называется **trivial closure**), и той, которая их захватывает (**non-trivial closure**).

Non-trivial closure не имеет своего символа значения функции в секции RODATA. Вместо этого её символ значения функции создаётся динамически в runtime вместе с захваченными переменными. Пример non-trivial closure:

go

```go
func main() {
    x := 1
    f := func(y int) int {
        return x + y
    }
    f(2)
    println(&f)
}
```

В ассемблерном коде нет символа значения функции вроде `main.func1·f`, а структура `funcval` создаётся динамически:

1. `MOVD $main.main.func1(SB), R1` — помещает адрес функции непосредственно в первое поле структуры `funcval`.
2. `MOVD $1, R1` — загружает значение 1 (значение `x`) и сохраняет его в следующей позиции структуры closure. Именно так closure «захватывает» переменную — она копирует значение в closure в момент её создания.

Весь процесс создания non-trivial closure можно резюмировать следующим псевдокодом:

go

```go
func main() {
    funcval := funcval{
        fn:   $main.func1,
        arg1: 1,
    }
    f := &funcval
    f(2)
}
```

Теперь посмотрим, что произойдёт, если изменить переменную `x` перед вызовом closure:

go

```go
func main() {
    x := 1
    f := func(y int) int {
        return x + y
    }
    x = 100
    f(2)
    println(&f)
}
```

В этом случае `x` захватывается по ссылке. Вместо хранения значения `$1` сохраняется адрес `x` в структуре closure. Функция closure затем разыменовывает этот адрес каждый раз, когда ей нужно получить значение.

Есть три условия, при которых компилятор Go захватывает по значению, а не по ссылке:

- Адрес переменной никогда не берётся.
- Переменная никогда не переприсваивается после захвата closure (включая внутри самой closure). Если она переприсваивается, она будет захвачена по ссылке.
- Размер переменной не превышает 128 байт.

Чтобы завершить этот раздел, рассмотрим более продвинутый пример:

go

```go
func returnAdder() func(int) int {
    x := 1
    return func(y int) int {
        x += y
        return x
    }
}
func main() {
    adder := returnAdder()
    println(adder(1)) // 2
    println(adder(2)) // 4
    println(adder(3)) // 7
}
```

Переменная `x` захватывается по ссылке, поскольку она изменяется внутри closure (`x += y`). Чтобы память `x` «оставалась живой», компилятор выделяет память в heap для `x`, которая переживёт stack frame функции. Ещё одно выделение в heap происходит для самой структуры closure:

go

```go
struct { F uintptr; X0 *int }
```

В функции `main` каждый вызов closure работает с одной и той же переменной `x`, выделенной в heap, через указатель, хранящийся в структуре closure. Вот почему значение накапливается: первый вызов возвращает 2 (1+1), второй — 4 (2+2), третий — 7 (4+3).

### Дизайн вариадических функций

Функция становится «вариадической», когда её последний параметр использует модификатор `...`. Это указывает компилятору собирать любое количество переданных значений (ноль, одно или несколько) в один slice.

Внутри функции вы работаете с этим параметром точно так же, как с любым другим slice:

go

```go
func sum(nums ...int) int {
    total := 0
    for _, v := range nums {
        total += v
    }
    return total
}
sum(1, 2, 3, 4, 5)
sum(1, 2)
```

Вариадические функции широко используются в стандартной библиотеке, например `fmt.Println` или встроенная функция `append`:

go

```go
func Println(a ...any) (n int, err error) {}
func append(slice []Type, elems ...Type) []Type {}
```

Если у вас уже есть slice и вы хотите передать его в вариадическую функцию, вам нужно использовать завершающий `...` для «взрыва» slice обратно в отдельные аргументы:

go

```go
var s = []int{1, 2, 3, 4, 5}
sum(s...)
```

При вызове вариадической функции:

- Компилятор упаковывает дополнительные аргументы в новый slice и передаёт его функции.
- Если вы вызываете функцию со slice и завершающим `...`, компилятор передаёт slice напрямую в функцию.

Синтаксис `func(slice...)` не «распаковывает» элементы slice — он передаёт сам slice, поэтому изменения элементов внутри функции будут видны снаружи:

go

```go
func sum(nums ...int) int {
    total := 0
    for _, v := range nums {
        total += v
    }
    nums[0] = 100
    return total
}
func main() {
    s := []int{1, 2, 3, 4, 5}
    fmt.Println(sum(s...)) // 15
    fmt.Println(s)         // [100 2 3 4 5]
}
```

Когда аргументы не передаются, создаётся `nil` slice, а не пустой slice.
