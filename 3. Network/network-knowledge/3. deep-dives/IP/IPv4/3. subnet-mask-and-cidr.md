# Subnet mask va CIDR

## Subnet mask nima uchun kerak?

IP address yolg'iz o'zi yetarli emas. Host qaysi tarmoqda ekanini bilish uchun **subnet mask** kerak.

Subnet mask IP addressni ikki qismga ajratadi:

1. **Network qismi** - qaysi subnet ekanini bildiradi.
2. **Host qismi** - shu subnet ichidagi qurilmani bildiradi.

Maskada:

```text
1 = network bit
0 = host bit
```

## Oddiy misol

```text
IP:    192.168.48.68
Mask:  255.255.255.0
CIDR:  /24
```

Binary ko'rinish:

```text
IP:    11000000.10101000.00110000.01000100
Mask:  11111111.11111111.11111111.00000000
       <---------- network ---------><--host-->
```

Demak:

```text
Network: 192.168.48.0
Host:    shu subnetdagi 68-host
```

`192.168.48.68/24` degani:

- `192.168.48.0` - network address
- `192.168.48.68` - host address
- `255.255.255.0` - subnet mask
- `/24` - 24 ta bit network uchun ajratilgan

## CIDR notation

CIDR - **Classless Inter-Domain Routing**.

CIDR IP address yonida prefix length yozish usuli:

```text
192.168.48.68/24
```

Bu yerda:

- `192.168.48.68` - IP address
- `/24` - birinchi 24 bit network qismi
- qolgan `32 - 24 = 8 bit` host qismi

## Prefix length va host bit

IPv4 32 bit. Prefix qancha katta bo'lsa, network qismi shuncha katta, host qismi shuncha kichik bo'ladi.

```text
Host bit = 32 - prefix
```

Masalan:

```text
/24 -> 32 - 24 = 8 host bit
/26 -> 32 - 26 = 6 host bit
/30 -> 32 - 30 = 2 host bit
```

Host bitlar subnet ichida nechta address bo'lishini belgilaydi:

```text
Jami address = 2^(host bitlar soni)
```

Odatdagi subnetlarda usable host:

```text
Usable host = 2^(host bitlar soni) - 2
```

Nega `-2`?

- 1 ta address network address uchun ketadi.
- 1 ta address broadcast address uchun ketadi.

Istisno:

- `/31` point-to-point linklarda 2 ta address sifatida ishlatilishi mumkin.
- `/32` bitta host yoki loopback route uchun ishlatiladi.

## Maskning decimal qiymatlari qayerdan keladi?

Mask octetlari binaryda ketma-ket `1`lardan keyin `0`lardan iborat bo'ladi.

Masalan:

```text
10000000 = 128
11000000 = 192
11100000 = 224
11110000 = 240
11111000 = 248
11111100 = 252
11111110 = 254
11111111 = 255
```

Shuning uchun subnet mask octetlarida odatda quyidagi qiymatlar uchraydi:

```text
0, 128, 192, 224, 240, 248, 252, 254, 255
```

Masalan `/27`:

```text
/27 = 24 ta to'liq bit + yana 3 bit
11111111.11111111.11111111.11100000
255.255.255.224
```

## Mask noto'g'ri bo'lsa nima bo'ladi?

Mask hostning "destination men bilan bir subnetdami?" degan qaroriga ta'sir qiladi.

Masalan:

```text
Host A: 192.168.1.10/24
Host B: 192.168.1.200/25
```

Bu yerda hostlar bir-birini turlicha talqin qilishi mumkin. Natijada ARP, gateway va routing bilan bog'liq noaniq muammolar chiqadi.

## Qisqa xulosa

Subnet mask IP addressni network va host qismlarga ajratadi. CIDR esa maskni qisqa yozish usuli. IPv4 troubleshootingda IP address bilan birga prefixni ham ko'rish shart, chunki aynan mask hostning lokalmi yoki gateway orqalimi degan qarorini belgilaydi.
