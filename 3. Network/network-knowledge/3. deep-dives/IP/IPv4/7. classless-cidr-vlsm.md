# Classless IPv4, CIDR va VLSM

## Classless model nima?

Classless modelda subnet mask IP classga bog'lanib qolmaydi.

Classful fikrlashda:

```text
192.x.x.x = Class C = /24
```

deyish mumkin edi.

Classless modelda esa bunday subnetlar normal:

```text
192.168.0.0/16
192.168.1.0/24
192.168.1.128/25
192.168.1.192/26
```

Ya'ni addressning birinchi octeti emas, **prefix length** subnet chegarasini belgilaydi.

## CIDR nima muammoni yechdi?

CIDR - **Classless Inter-Domain Routing**.

CIDR foydalari:

- IP addressni tejaydi.
- Subnetlarni real ehtiyojga moslash imkonini beradi.
- Routing table'ni summarization orqali kichraytiradi.
- VLSM ishlatishga imkon beradi.
- Classful default maskga bog'liqlikni olib tashlaydi.

## Summarization

Summarization - bir nechta kichik route'ni bitta umumiy route bilan ifodalash.

Misol:

```text
192.168.0.0/16
```

Bu bitta summary route bo'lib, quyidagilarni qamrab oladi:

```text
192.168.0.0/24
192.168.1.0/24
192.168.2.0/24
...
192.168.255.0/24
```

Routing table uchun bu juda foydali: ko'p mayda route o'rniga bitta umumiy route yoziladi.

## VLSM nima?

VLSM - **Variable Length Subnet Mask**.

VLSM bir address block ichida turli subnetlarga turli mask berish imkonini beradi.

Masalan tashkilotga quyidagilar kerak:

| Segment | Kerakli host | Mos subnet |
|---|---:|---|
| LAN A | 100 host | /25 |
| LAN B | 50 host | /26 |
| LAN C | 20 host | /27 |
| Point-to-point | 2 host | /30 yoki /31 |

Classful modelda bularni juda noqulay taqsimlash kerak bo'lardi. VLSM bilan esa har segmentga kerakli hajmga yaqin subnet ajratiladi.

## CIDR va VLSM farqi

| Tushuncha | Ma'nosi |
|---|---|
| CIDR | Classless prefix notation va routing yondashuvi |
| VLSM | Bir address blockni turli uzunlikdagi subnetlarga bo'lish |
| Summarization | Bir nechta route'ni bitta umumiy prefixga yig'ish |

Ular bir-biriga yaqin, lekin aynan bir narsa emas.

## Classless routingda eng muhim qoida

Router destination IP uchun routing table'dan **eng aniq prefix**ni tanlaydi.

Bu qoida:

```text
Longest Prefix Match
```

deyiladi.

Misol routing table:

```text
0.0.0.0/0          via 192.168.1.1
10.0.0.0/8         via 172.16.1.1
10.10.0.0/16       via 172.16.2.1
10.10.20.0/24      via 172.16.3.1
```

Destination:

```text
10.10.20.55
```

Tanlanadigan route:

```text
10.10.20.0/24
```

Chunki `/24` `/16`, `/8` va `/0` ga qaraganda aniqroq.

## Classless yondashuvdagi xatolik

Eng ko'p uchraydigan xato:

```text
192.168.1.10 ko'rdim, demak /24.
```

Bu noto'g'ri. Quyidagilarning hammasi mumkin:

```text
192.168.1.10/16
192.168.1.10/24
192.168.1.10/25
192.168.1.10/30
```

IP address bir xil, lekin subnetlar boshqacha.

## Qisqa xulosa

Classless IPv4 da subnet chegarasini class emas, prefix length belgilaydi. CIDR addresslarni tejash va route'larni umumlashtirish uchun ishlatiladi. VLSM esa bir address blockni real ehtiyojga mos turli subnetlarga bo'lish imkonini beradi.
