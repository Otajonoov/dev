# Classful IPv4

## Classful model nima?

IPv4 boshida classful model bilan ishlagan. Addressning birinchi octeti qaysi class ekanini bildirgan va mask shu classga bog'langan.

Bugungi amaliy tarmoqlarda classful model asosiy yondashuv emas, lekin CCNA va tarixiy tushuncha uchun muhim.

## Class A, B, C, D, E

| Class | Range | Default mask | Maqsad |
|---|---|---|---|
| A | 1.0.0.0 - 126.255.255.255 | 255.0.0.0 (/8) | Juda katta network |
| B | 128.0.0.0 - 191.255.255.255 | 255.255.0.0 (/16) | O'rta network |
| C | 192.0.0.0 - 223.255.255.255 | 255.255.255.0 (/24) | Kichik network |
| D | 224.0.0.0 - 239.255.255.255 | mask yo'q | Multicast |
| E | 240.0.0.0 - 255.255.255.255 | mask yo'q | Reserved |

## Nega 127.x.x.x oddiy Class A emas?

Class A jadvalida 1-126 ko'rsatiladi, chunki:

```text
127.0.0.0/8
```

loopback uchun reserved.

Masalan:

```text
127.0.0.1
```

localhost sifatida ishlatiladi.

## Classful model qanday fikrlaydi?

Classful fikrlashda mask address classidan kelib chiqadi:

```text
10.1.2.3       -> Class A -> /8
172.16.1.10    -> Class B -> /16
192.168.1.10   -> Class C -> /24
```

Bu tarixiy yondashuv. Zamonaviy routingda bunday xulosa qilish xavfli, chunki hozir mask istalgan prefix bo'lishi mumkin.

Masalan:

```text
10.10.10.0/24
172.16.10.0/27
192.168.0.0/16
```

Bular classful default masklarga mos kelmaydi, lekin amalda normal classless subnetlardir.

## Classful modelning muammosi

Classful model addresslarni real ehtiyojga mos taqsimlay olmadi.

Muammolar:

- Class A juda katta: bitta tashkilotga millionlab address berilib ketgan.
- Class B ham ko'p holatda ortiqcha katta.
- Class C esa ba'zan juda kichik.
- IP addresslar isrof bo'lgan.
- Routing table'lar kattalashib borgan.

Misol:

```text
Tashkilotga 2,000 ta host kerak.
```

Class C:

```text
/24 = 254 usable host
```

yetmaydi.

Class B:

```text
/16 = 65,534 usable host
```

juda ortiqcha.

Shu sababli CIDR va classless routing paydo bo'ldi.

## Class D va E

Class D:

```text
224.0.0.0 - 239.255.255.255
```

multicast uchun ishlatiladi. Bu hostga beriladigan oddiy unicast address emas.

Class E:

```text
240.0.0.0 - 255.255.255.255
```

reserved/experimental maqsadlar uchun ajratilgan.

## Classful xatolik

Noto'g'ri fikr:

```text
10.x.x.x doim /8 bo'ladi
192.168.x.x doim /24 bo'ladi
```

To'g'ri fikr:

```text
Mask yoki CIDR ko'rsatilmasa, subnet chegarasini aniq aytib bo'lmaydi.
```

## Qisqa xulosa

Classful IPv4 - tarixiy model. U A/B/C classlarga default mask bog'laydi, lekin bugungi tarmoqlar classless ishlaydi. Shuning uchun classlarni bilish kerak, ammo real konfiguratsiyada doim CIDR prefix yoki subnet maskga qaraladi.
