# IP addressing asoslari

## IPv4 nima?

IPv4 - **Internet Protocol version 4**. U OSI modelidagi **Network Layer** darajasida ishlaydi va qurilmalarga mantiqiy address beradi.

Network Layer'ning ikki asosiy vazifasi bor:

1. **IP addressing** - har bir host yoki router interface'ga IP address berish.
2. **Routing** - packetni qaysi yo'l orqali destination network'ga yetkazishni tanlash.

Oddiy ajratish:

| Tushuncha | Qayerda ishlaydi | Vazifasi |
|---|---|---|
| MAC address | Local network segment ichida | Frame'ni lokal tarmoqda yetkazish |
| IP address | Networklar orasida | Packetni source'dan destination'ga yo'naltirish |
| Switch | Bir LAN ichida | MAC table asosida frame uzatish |
| Router | Networklar orasida | IP route asosida packet uzatish |

IPv4 bugungi Internetda hali ham juda keng ishlatiladi. IPv6 mavjud, lekin NAT, private address va subnetting sababli IPv4 amalda uzoq vaqt davomida saqlanib qoldi.

## IPv4 address necha bit?

IPv4 address - **32 bitli** manzil.

```text
2^32 = 4,294,967,296 ta nazariy address
```

Bu "nazariy" son, chunki hamma addresslar hostlarga berilmaydi. Ayrim range'lar private, loopback, multicast, documentation yoki reserved maqsadlarda ishlatiladi.

## Dotted decimal notation

IPv4 odatda nuqtalar bilan ajratilgan decimal ko'rinishda yoziladi:

```text
192.168.48.68
```

Bu aslida 4 ta 8 bitli qismdan iborat:

```text
192      168      48       68
11000000.10101000.00110000.01000100
```

Har bir qism **octet** deyiladi.

```text
8 bit + 8 bit + 8 bit + 8 bit = 32 bit
```

## Octet nima?

Octet - 8 bit.

8 bit bilan yoziladigan eng kichik qiymat:

```text
00000000 = 0
```

8 bit bilan yoziladigan eng katta qiymat:

```text
11111111 = 255
```

Shuning uchun IPv4 addressdagi har bir octet:

```text
0 dan 255 gacha
```

bo'lishi mumkin.

To'g'ri addresslar:

```text
10.0.0.1
192.168.1.254
8.8.8.8
```

Noto'g'ri addresslar:

```text
192.168.1.300
10.0.0.-1
1.2.3.4.5
```

## IP addressning o'zi yetarlimi?

Yo'q. IPv4 addressni to'g'ri tushunish uchun IP bilan birga **subnet mask** yoki **CIDR prefix** ham kerak.

Masalan:

```text
192.168.1.10
```

bu addressning qaysi subnetga tegishli ekanini aniq aytish uchun mask kerak:

```text
192.168.1.10/24
192.168.1.10/25
192.168.1.10/26
```

Bu uchala holatda host IP bir xil, lekin subnet chegarasi boshqa-boshqa bo'lishi mumkin.

## Host va interface farqi

IP address odatda "kompyuterga beriladi" deb aytiladi, lekin aniqroq aytganda IP address **interface**ga beriladi.

Masalan routerda 3 ta interface bo'lsa:

```text
G0/0: 192.168.1.1/24
G0/1: 192.168.2.1/24
G0/2: 10.10.10.1/24
```

Routerning o'zi bitta qurilma, lekin har bir network tomoni uchun alohida IP addressga ega.

## IPv4 address va packet

IPv4 packet ichida source va destination IP addresslar bo'ladi:

```text
Source IP:      192.168.1.10
Destination IP: 93.184.216.34
```

Routerlar packetni destination IP bo'yicha yo'naltiradi. Ethernet frame ichidagi MAC addresslar esa har hop'da o'zgarishi mumkin.

Muhim farq:

| Layer | Address | Qanday ishlaydi |
|---|---|---|
| Layer 2 | MAC address | Lokal link ichida yetkazish |
| Layer 3 | IP address | Networklar orasida yo'naltirish |

## Qisqa xulosa

IPv4 addressing - hostlarni mantiqiy tarmoq manzillari bilan belgilash tizimi. Address 32 bitdan iborat, 4 ta octetga bo'linadi, har octet 0-255 oralig'ida bo'ladi. Lekin IP addressni to'liq tushunish uchun doim mask yoki CIDR prefix bilan birga qarash kerak.
