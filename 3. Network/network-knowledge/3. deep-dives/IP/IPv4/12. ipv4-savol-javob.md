# IPv4 savol-javob

Bu fayldagi savollar IPv4 ni yodlash uchun emas, mohiyatini tushunish uchun yozilgan.

## Nega IPv4 address 255 gacha?

IPv4 address 4 ta octetdan iborat. Har bir octet 8 bit.

8 bitning eng katta binary qiymati:

```text
11111111
```

Decimalga o'tkazsak:

```text
128 + 64 + 32 + 16 + 8 + 4 + 2 + 1 = 255
```

Shuning uchun har bir octet `0` dan `255` gacha bo'ladi. `256` bo'lishi uchun 9-bit kerak bo'lardi, lekin octet faqat 8 bit.

## Nega IPv4 4 ta octetdan iborat?

IPv4 address 32 bit qilib belgilangan.

32 bitni o'qish va yozish qulay bo'lishi uchun 4 ta 8 bitli octetga bo'lib yozamiz:

```text
8 + 8 + 8 + 8 = 32
```

Masalan:

```text
192.168.1.10
```

kompyuter uchun aslida:

```text
11000000.10101000.00000001.00001010
```

## IP addressning o'zi nega yetarli emas?

Chunki IP address qaysi qismi network, qaysi qismi host ekanini aytmaydi.

Masalan:

```text
192.168.1.10
```

bu quyidagicha bo'lishi mumkin:

```text
192.168.1.10/24
192.168.1.10/25
192.168.1.10/30
```

IP bir xil, lekin subnet chegarasi boshqa. Shuning uchun IP bilan birga mask yoki CIDR kerak.

## Subnet mask aslida nima qiladi?

Mask IP addressni ikkiga ajratadi:

```text
network qismi + host qismi
```

Maskdagi `1` bitlar networkni, `0` bitlar hostni bildiradi.

```text
IP:    192.168.1.10
Mask:  255.255.255.0
CIDR:  /24
```

Bu "birinchi 24 bit network, qolgan 8 bit host" degani.

## `/24` degani nima?

`/24` - IP addressning birinchi 24 biti network uchun ajratilgan.

IPv4 32 bit bo'lgani uchun:

```text
32 - 24 = 8 host bit
```

8 host bit bilan:

```text
2^8 = 256 ta address
```

Lekin odatda usable host:

```text
256 - 2 = 254
```

bo'ladi.

## Nega usable host hisobida `-2` qilinadi?

Odatdagi subnetlarda 2 ta address hostga berilmaydi:

1. **Network address** - subnetning nomi.
2. **Broadcast address** - subnetdagi hammaga yuborish addressi.

Masalan `192.168.1.0/24`:

```text
Network:   192.168.1.0
Broadcast: 192.168.1.255
Hosts:     192.168.1.1 - 192.168.1.254
```

Shuning uchun usable hostlar soni:

```text
2^host_bits - 2
```

Istisno: `/31` point-to-point linklarda boshqacha ishlatiladi.

## Network address nima uchun kerak?

Network address subnetni nomlash uchun kerak.

Masalan:

```text
192.168.10.0/24
```

bu bitta host emas, balki butun subnetni bildiradi.

Routing table'da ham route subnet nomi bilan yoziladi:

```text
192.168.10.0/24 via 10.0.0.1
```

Bu "192.168.10.0/24 subnetiga borish uchun 10.0.0.1 orqali ket" degani.

## Broadcast address nima uchun kerak?

Broadcast address subnet ichidagi barcha hostlarga bitta packet yuborish uchun kerak.

Masalan:

```text
192.168.1.255
```

`192.168.1.0/24` subnetining directed broadcast addressi.

DHCP kabi jarayonlarda broadcast muhim, chunki client hali server qayerda ekanini bilmaydi.

## Router broadcastni nega boshqa subnetga o'tkazmaydi?

Broadcast lokal subnet uchun mo'ljallangan. Agar router broadcastni hamma subnetlarga o'tkazaversa, tarmoqda ortiqcha traffic va xavfsizlik muammolari ko'payadi.

Shuning uchun routerlar odatda broadcast domain chegarasi hisoblanadi.

## Host destination boshqa subnetda ekanini qanday biladi?

Host o'z IP/maskini destination IP bilan solishtiradi.

Masalan:

```text
Host:        192.168.1.10/24
Destination: 192.168.1.20
```

Ikkalasi `192.168.1.0/24` ichida, demak lokal.

```text
Host:        192.168.1.10/24
Destination: 192.168.2.20
```

Destination boshqa subnetda, demak default gateway kerak.

## Default gateway nima uchun kerak?

Host faqat o'z lokal subnetidagi qurilmalarga to'g'ridan-to'g'ri ARP qila oladi.

Boshqa subnetga borish uchun host packetni routerga beradi. Shu router interface host configuration'da default gateway bo'ladi.

```text
IP:      192.168.1.10/24
Gateway: 192.168.1.1
```

`192.168.1.1` hostning boshqa subnetlarga chiqish eshigi.

## Gateway nega host bilan bir subnetda bo'lishi kerak?

Host gateway MAC addressini ARP orqali topadi. ARP faqat lokal subnet/broadcast domain ichida ishlaydi.

Shuning uchun:

```text
Host:    192.168.1.10/24
Gateway: 192.168.2.1
```

noto'g'ri. Host `192.168.2.1` ga lokal ARP qila olmaydi.

## ARP nima uchun kerak?

IPv4 packetni Ethernet orqali yuborish uchun MAC address kerak.

ARP savol beradi:

```text
192.168.1.1 kim? MAC addressingni ayt.
```

Gateway javob beradi:

```text
192.168.1.1 menman, MAC: aa:bb:cc:dd:ee:ff
```

Keyin host IP packetni gateway MAC'iga frame qilib yuboradi.

## Boshqa subnetga ketganda destination IP o'zgaradimi?

Yo'q. Destination IP oxirgi manzil bo'lib qoladi.

Masalan:

```text
Source IP:      192.168.1.10
Destination IP: 93.184.216.34
```

PC packetni gateway MAC'iga yuboradi, lekin IP destination baribir `93.184.216.34`.

Har hop'da MAC addresslar o'zgarishi mumkin, IP addresslar esa odatda o'zgarmaydi. NAT bo'lsa, source yoki destination IP o'zgarishi mumkin.

## Router route'ni qanday tanlaydi?

Router destination IP uchun routing table'dan eng aniq route'ni tanlaydi.

Bu qoida:

```text
Longest Prefix Match
```

Misol:

```text
0.0.0.0/0
10.0.0.0/8
10.10.0.0/16
10.10.20.0/24
```

Destination `10.10.20.55` bo'lsa, router `/24` route'ni tanlaydi, chunki u eng aniq.

## `0.0.0.0/0` nimani bildiradi?

`0.0.0.0/0` default route.

`/0` degani network bit yo'q, ya'ni hamma destination addresslar mos tushadi.

Lekin routingda aniqroq route bo'lsa, default route emas, aniqroq route ishlatiladi.

## `0.0.0.0` va `0.0.0.0/0` bir xilmi?

Yo'q, kontekstiga qarab farq qiladi.

`0.0.0.0`:

- host hali address olmaganini bildirishi mumkin;
- service "hamma interface'da listen qil" degan ma'noda ishlatilishi mumkin.

`0.0.0.0/0`:

- default route;
- barcha destinationlarni qamrab oluvchi prefix.

## `127.0.0.1` nima uchun ishlatiladi?

`127.0.0.1` localhost. Bu hostning o'ziga murojaat qilish addressi.

```bash
ping 127.0.0.1
```

Bu packet fizik networkga chiqmaydi, OS ichida loopback orqali qaytadi.

## Private IP Internetda nega to'g'ridan-to'g'ri ishlamaydi?

Private IP range'lar global Internetda unique emas.

Masalan:

```text
192.168.1.10
```

minglab uy tarmoqlarida bo'lishi mumkin. Internet routerlari bunday addresslarni global route qilmaydi.

Shuning uchun private host Internetga chiqish uchun NAT yoki tunnel ishlatadi.

## NAT IPv4 muammosini qanday yengillashtirdi?

IPv4 addresslar soni cheklangan. NAT bitta public IP orqasida ko'p private host ishlashiga imkon berdi.

Masalan:

```text
192.168.1.10:51515 -> 203.0.113.10:40001
192.168.1.11:51516 -> 203.0.113.10:40002
```

Router portlar orqali qaysi ichki host qaysi connection ochganini eslab turadi.

## Classful va classless farqi nimada?

Classful model addressning birinchi octetiga qarab default mask tanlaydi:

```text
Class A -> /8
Class B -> /16
Class C -> /24
```

Classless modelda mask istalgan prefix bo'lishi mumkin:

```text
10.10.10.0/24
192.168.0.0/16
192.168.1.128/25
```

Bugungi amaliy routing classless ishlaydi.

## Nega "192.168.x.x doim /24" deyish xato?

Chunki `192.168.x.x` private range ekanini bildiradi, subnet maskni emas.

Quyidagilarning hammasi mumkin:

```text
192.168.1.10/16
192.168.1.10/24
192.168.1.10/27
```

Subnet chegarasini doim mask yoki CIDR belgilaydi.

## `/32` nima?

`/32` - 32 bitning hammasi network qismi, host bit yo'q.

Bu bitta aniq IP addressni bildiradi:

```text
192.168.1.10/32
```

Loopback interface, host route yoki firewall qoidasida ko'p ishlatiladi.

## `/31` da nega 2 ta address ishlatilishi mumkin?

Odatdagi subnetlarda network va broadcast address ajratiladi. Lekin point-to-point linkda faqat ikki uch bor va broadcastga ehtiyoj yo'q.

Shuning uchun `/31` ikki router orasidagi point-to-point linklarda 2 ta address sifatida ishlatilishi mumkin.

## Subnet mask IPv4 header ichida yuradimi?

Yo'q. IPv4 header ichida source IP va destination IP bor, lekin subnet mask yo'q.

Mask:

- host configuration'da bo'ladi;
- router routing table'da prefix sifatida bo'ladi.

Packetning o'zi maskni olib yurmaydi.

## TTL nima uchun kerak?

TTL packet routing loopda cheksiz aylanib qolmasligi uchun kerak.

Har router TTL'ni 1 taga kamaytiradi. TTL 0 bo'lsa, packet tashlab yuboriladi.

Traceroute ham shu mexanizmdan foydalanib hoplarni aniqlaydi.

## Fragmentation nima uchun muammo bo'lishi mumkin?

Fragmentation packetni kichik bo'laklarga bo'ladi. Bu:

- performance'ni pasaytirishi mumkin;
- firewall tekshiruvini murakkablashtiradi;
- ayrim pathlarda ICMP bloklansa, Path MTU Discovery buziladi.

Shuning uchun zamonaviy tarmoqlarda packet hajmini path MTU'ga moslash muhim.

## DHCP Discover nega broadcast bo'ladi?

Client boshida:

- o'z IP'sini bilmaydi;
- DHCP server IP'sini bilmaydi;
- gatewayni bilmaydi.

Shuning uchun u:

```text
Source IP:      0.0.0.0
Destination IP: 255.255.255.255
```

bilan broadcast yuboradi.

## DNS packetni yetkazadimi?

Yo'q. DNS faqat hostname'ni IP addressga aylantiradi.

Masalan:

```text
example.com -> 93.184.216.34
```

Packetni yetkazish routing vazifasi.

## IP address publicmi yoki privatemi, nega bu muhim?

Private IP Internetda global route qilinmaydi. Public IP esa Internetda unique va route qilinadi.

Troubleshootingda bu juda muhim:

- private IP bilan tashqi Internetdan to'g'ridan-to'g'ri hostga kira olmaysiz;
- NAT, port forwarding yoki VPN kerak bo'lishi mumkin;
- edge router private source/destination route'larni filtrlashi kerak.

## `169.254.x.x` ko'rsam nimani o'ylashim kerak?

Ko'pincha DHCP ishlamaganini.

Host DHCP serverdan address ololmasa, o'ziga link-local/APIPA address berishi mumkin:

```text
169.254.0.0/16
```

Bunda lokal segmentda cheklangan aloqa bo'lishi mumkin, lekin normal gateway va Internet ishlamasligi ehtimoli yuqori.

## IPv4 encryption qiladimi?

Yo'q. IPv4 packetni addresslash va routing uchun ishlatiladi, lekin u o'zi encryption bermaydi.

Xavfsizlik uchun:

- TLS;
- SSH;
- IPsec;
- VPN;
- application encryption

ishlatiladi.

## IPv4 ni tushunish uchun eng muhim mental model nima?

Har packetda 3 bosqichni o'ylash kerak:

1. **Host qarori** - destination lokal subnetdami yoki gateway kerakmi?
2. **L2 yetkazish** - next-hop MAC address ARP orqali topiladimi?
3. **L3 routing** - router destination IP uchun eng aniq route'ni tanlaydimi?

Shu model IPv4 troubleshootingning katta qismini tushuntirib beradi.
