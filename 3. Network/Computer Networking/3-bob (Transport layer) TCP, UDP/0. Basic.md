
Transport layer network arxitekturasining markaziy qismi bo'lib, application layer va network layer o'rtasida joylashgan. U turli hostlarda ishlab turgan application processlari uchun bevosita aloqa xizmatlarini taqdim etishda muhim rol o'ynaydi.

Transport layer protokoli turli hostlarda bajarilayotgan application processlar o'rtasida **logical connection** ta'minlaydi. Bu mantiqiy ulanish ilovalar nuqtai nazaridan jarayonlarni bevosita bog'laydigan kanal kabi ko'rinadi, hatto xostlar sayyoraning turli burchaklarida joylashgan va ular orasidagi haqiqiy aloqa ko'plab routerlar va turli xil aloqa liniyalari orqali amalga oshirilsa ham.

==Transport layer==: processlar o'rtasida logical connection
==Network layer==: hostlar o'rtasida logical connection

Tarmoq routerlari faqat tarmoq qatlami darajasida datagramma maydonlarini qayta ishlaydi - ular datagrammaga inkapsulatsiya qilingan transport qatlami segmentining maydonlarini tekshirmaydi.

##### Transport layer qanday ishlaydi?

**Sender tomonda:**
- Application layer xabarlarini **segment** larga aylantiradi
- Kerak bo'lganda xabarlarni bo'laklarga ajratadi
- Har bir bo'lakka ==transport== layer ==header== qo'shadi
- Segmentni network layerga uzatadi

**Receiver tomonda:**
- Network layer datagramdan transport segmentini ajratib oladi
- Transport layer segmentni qayta ishlaydi
- Ma'lumotlarni qabul qiluvchi application'ga yetkazadi


Transport layer taqdim eta oladigan xizmatlar network layer imkoniyatlari bilan cheklanadi, lekin...

**Qiziq fakt:** Transport layer ba'zi xizmatlarni network layer qo'llab-quvvatlamasa ham taqdim eta oladi!

Masalan:

1. **Reliable data transfer**: Network layer ishonchsiz bo'lsa ham (paketlar yo'qolishi mumkin), transport layer ishonchli xizmat yarata oladi
2. **Security**: Shifrlash orqali xavfsizlikni ta'minlashi mumkin
3. **Multiplexing/Demultiplexing** - process'lar o'rtasida ma'lumot almashinuvi
4. **Error detection** - header'da xatolarni aniqlash maydoni
