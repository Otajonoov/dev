**_TCP slow start_** bu TCP ning bir xususiyati (feature) bo`lib tarmoqdagi oqimni boshqarishga yordam beradi.

## **U qanday ishlaydi ?**

TCP handshakedan so`ng TCP tarmoq orqali juda kichik paketlarni jo`natishni boshlaydi.Agarda server ularni yetib kelganini tezda tasdiqlasa ,paketning hajmi sekinlik bilan oshirib borilaveradi .Bu holatni toki server paketlarga acknowledgement yuborolmaydigan yoki boshqa muammolar kelib chiqqancha davom ettiramiz shu payt TCP bu yetarli deydi va shu hajmida bir xil paketlar uzatilaveradi.

### Afzallik

Buning yaxshi tomoni siz yo`lda ya’ni tarmoqda paketlar yo`qolishi va boshqa muammolarga duch kelmagan holda o`zingizni qoniqtiradigan **_performancega_** chiqa olasiz.

### Muammo

Agarda [**_handshakedan_**](https://medium.com/@habibovulugbek22/tcp-handshake-dee5418cef95) so`ng darhol juda katta hajmdagi malumot yubormoqchi bo`lsangiz siz performance pastligini his qilishingiz mumkin.Masalan sizda 5GB malumotni darhol uzatish kerak buni server va foydalanuvchi qurilmalari tomondan iloji bor ammo TCP slow start tufayli bu jarayon sekinlashadi va TCP qizib olishi yani kattaroq packetlarni uzatishni boshlashi bir muncha vaqt talab etadi.Bu muammo qachonki siz handshakedan so`ng darhol juda katta hajmdagi POST so`rovlar jo`natganda kelib chiqishi mumkin va har doim faqatgina boshlanishida .Oddiy GET so`rovlarda buni siz his qilmaysiz.

### Yechim

Bazi providerlar TCP connection ochib ularda shunchaki malumot almashib turishadi qachonki yangi request kelganda maummoga uchramaslik uchun.

### Xulosa

Xulosa qilib aytadigan bo`lsak TCPning bu xususiyati so`rovlarga javobni sekinlashtiradi ammo biz server yoki tarmoqni birdani katta malumot bilan qulatib yuborishimizni oldini oladi.
