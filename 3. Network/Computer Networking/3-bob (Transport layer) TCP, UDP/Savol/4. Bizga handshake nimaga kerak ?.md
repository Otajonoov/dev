## Bizga handshake nimaga kerak ?

TCP bizga so`rovlarni hammasi yetiborishiga kafolat beradi buni qanday amalga oshiradi .So`rov yuboriladi va serverdan qabul qilinganligi haqida ACK qabul qiladi .Agar ACK qaytib kelmasa so`rov qayta uzatiladi.

> [SYN](https://medium.com/@habibovulugbek22/tcp-va-udp-8ee56341c713) — bu aloqa o`rnatish uchun uzatilgan so`rov.  
> ACK — bu uzatuvchi uzatgan ma’lumotni qabul qildim degani .(qayta uzatilmasin)

Xo`sh foydalanuvchi aynan qaysi so`rovga ACK kelganini qayerdan biladi shu muammoni yechish uchun ular handshake qilishlari yani umumiy bir sequence number yani packetlarni ajratib oluvchi raqamlarga kelishib olishlari zarur.

## Handshake qanday o`rnatiladi ?

Foydalanuvchi get so`rovini yubordi bu holatda foydalanuvchi va server o`rtasida aloqa yani handshake o`rnatiladi va bundan so`ng get so`rovi yuboriladi . Foydalanuvchi serverga [SYN](https://medium.com/@habibovulugbek22/tcp-va-udp-8ee56341c713) sequnce number uzatadi masalan 287 _(bu har doim bir xil sondan boshlanmaydi sababi esa ortiqcha hujumlardan himoyalanish uchun)_ va server buni qabul qiladi va unga qaytib SYN sequnce number hamda ACK yuborilgan sequnce numberga va packet uzunligi qiymati qo`shib uzatiladi (_handshake jarayonida bu 1 byteni tashkil qiladi_ ) so`rovi foydalanuvchiga uzatiladi. Foydalanuvchi bunga javoban ACK paketi uzatiladi server sequence numberga 1 ni qo`shib.Va bu jarayon **_3-way handshake_** deb ataladi.

Bundan so`ng server haqiqiy so`rovlarni qabul qilishni boshlaydi GET so`rovini va bunga javoban ACK packeti qaytariladi ya’ni men so`rovni qabul qildim degani.
