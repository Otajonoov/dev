
Odatda client server bilan muloqotni boshlashdan oldin handshake qilib yani bir xil sequence number (paketning bir xil id)ga keltirib olishi zarur.Bu holat qanday amalga oshadi client Serverga SYN yuboradi.Bu paketning ichida bu so`rov clientning qaysi ip adres va qaysi portidan chiqayotganligi hamda serverdagi qaysi ip adres hamda portga borishi kiritilgan bo`ladi . Bundan so`ng server clientga qayta SYN , ACK jo`natiladi bu paket ham yuboruvchi server ning ip adres va porti hamda qabul qiluvchi clientning ip adres va porti biriktirilgan bo`ladi. Bundan so`ng client Tcp connectionni yopadi. Bu holatda server qayta ACK kutayotgan bo`ladi va qabul qila olmaydi buning natijasida .TCPning yektazib berishga bo`lgan kafolati ishga tushadi va qayta qayta client bilan bog`lanishga harakat qiladi .Va malum muddatdan so`ng server ham connectionni uzadi .Shu holatga **TCP half open** holati deyiladi.Bu odatda serverning qaysi portlari ochiqligini bilish uchun ham foydalaniladi.Bu holatda bir xil ip adresdan qayta qayta amalga oshirilsa ip adres bloklanish ehtimoli katta.


### SYN Flood amalga oshirilishi .

Hujum qiluvchi 1- bo`lib TCP half-open orqali serverning ochiq portini aniqlab olishga harakat qiladi.Port aniqlangandan so`ng TCP ning paketdagi ip adresni o`zgartirish orqali yana qayta so`rov yuboradi.Bu holatda server javobidan so`nga yana ACK kutishni boshlaydi va qabul qilmaydi va TCP half opendagi holatga keladi.Bu agar 1 marta bo`lsa serverga tasir bilinmaydi .Agar half-open holatlar soni oshib boraversa server TCP connectionlar haqida malumotlarni saqlab qolganligi uchun xotira bo`yicha muammolar kelib chiqadi . Bu holatni odatda DDosni oldini oluvchi xizmatlar himoyalaydi.
