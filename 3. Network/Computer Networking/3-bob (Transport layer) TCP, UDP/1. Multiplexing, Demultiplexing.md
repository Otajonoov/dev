
 Transport layer'ning muhim vazifalaridan biri - **multiplexing** va **demultiplexing**.
 
 **Multiplexing/Demultiplexing** - process'lar o'rtasida ma'lumot almashinuvi
 
Transport layer'ning muhim vazifasi - host'lar o'rtasidagi connection'ni process'lar darajasiga "kengaytirish".

**Multiplexing** - turli socket'lardan kelgan ma'lumotlarni segment'larga yig'ish

**Demultiplexing** - kelgan segment'larni tegishli socket'ga yo'naltirish 

**Receiving host'da:**
- Transport layer network layer'dan segment'larni qabul qiladi
- Bu segment'larni tegishli application process'ga yetkazadi

**Misol:** Kompyuteringizda bir vaqtda:
- Web browser (HTTP)
- FTP session
- Ikki Telnet session

Transport layer kelgan ma'lumotlarni to'rt process'dan qaysi biriga yuborishni qanday biladi?

Process'lar **socket**'lar orqali tarmoq bilan aloqa qiladi. Har bir socket o'zining unikal identifikatoriga ega:

- UDP socket - 2 ta parametr bilan aniqlanadi
- TCP socket - 4 ta parametr bilan aniqlanadi

Har bir transport layer segment'ida maxsus maydonlar bor:
- **Source port number** (16 bit, 0-65535)
- **Destination port number** (16 bit, 0-65535)

**Port raqamlari:**
- 0-1023: Well-known ports (HTTP=80, FTP=21)
- 1024-65535: Dynamic ports

Demultiplexing jarayoni
1. Segment keladi
2. Transport layer destination port'ni tekshiradi
3. Segment tegishli socket'ga yo'naltiriladi
4. Ma'lumot process'ga uzatiladi
---

#### 1. UDP Multiplexing/Demultiplexing

```python
clientSocket = socket(socket.AF_INET, socket.SOCK_DGRAM)
```

Transport layer avtomatik port raqam beradi (1024-65535 oralig'idan).
Yoki o'zimiz belgilashimiz mumkin:
```python
clientSocket.bind(('', 19157))
```

UDP socket **2 ta parametr** bilan aniqlanadi:
- Destination IP address
- Destination port number

**Muhim:** Agar ikki UDP segment turli source'lardan kelsa ham, bir xil destination IP va port'ga ega bo'lsa, **bitta socket'ga** yo'naltiriladi!

Source port nima uchun kerak?
	Source port "return address" vazifasini bajaradi. Server javob yuborishda source va destination port'larni almashtiradi.

---

#### 2. TCP Multiplexing/Demultiplexing

TCP socket **4 ta parametr** bilan aniqlanadi:
- Source IP address
- Source port number
- Destination IP address
- Destination port number

**Farq:** TCP'da har xil source'dan kelgan segment'lar **turli socket'larga** yo'naltiriladi!

TCP Connection o'rnatish jarayoni
1. Server "welcoming socket"da kutadi (masalan, port 12000)
2. Client connection request yuboradi
3. Server yangi connection socket yaratadi
4. To'rtta parametr bilan identifikatsiya qilinadi


Bir web server bir vaqtda ko'p TCP connection'larni qo'llab-quvvatlashi mumkin:
- Har bir connection uchun alohida socket
- Har bir socket 4 parametr bilan identifikatsiya qilinadi
