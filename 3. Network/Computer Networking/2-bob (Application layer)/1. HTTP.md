==**HTTP (HyperText Transfer Protocol)**== - bu aplication layerdagi protokol bo'lib, u quyidagilarni belgilaydi:
- Client va server qanday xabar almashadi
- Xabarlarning tuzilishi qanday bo'ladi
- Qachon va qanday tartibda xabarlar yuboriladi

- **TCP ustida ishlaydi** - HTTP o'zi ma'lumotlarning yetib borishini kafolatlamaydi, bu ishni TCP protokoli bajaradi.
- **Stateless** - server mijoz haqida hech qanday ma'lumot saqlamaydi. Agar bir mijoz bir xil ob'ektni ikki marta so'rasa ham, server buni bilmaydi va ob'ektni qayta yuboradi.
- 
HTTP protokoli TCP'ga tayanadi va holatni saqlamaydi (stateless). Bu muhim konsepsiya. Ulanishlar bo'yicha esa non-persistent ulanishlar har bir obyekt uchun alohida TCP ulanish talab qiladi, persistent ulanishlar esa bir nechta obyektlarni bitta ulanish orqali uzatish imkonini beradi.

#### Ulanish turlari: 

##### 1. ==Non-Persistent== (Vaqtinchalik)
-  Har bir obyekt uchun alohida TCP ulanish ochiladi.
-  Ob'ekt uzatilgandan keyin ulanish yopiladi
-  11 ta ob'ektli sahifa uchun 11 ta alohida TCP ulanish kerak
- **Jarayon bosqichlari** (bitta ob'ekt uchun):

**Jarayon bosqichlari** (bitta ob'ekt uchun):
1. Mijoz TCP ulanish o'rnatadi (port 80)
2. HTTP so'rov yuboradi
3. Server ob'ektni topib, javobda yuboradi
4. TCP ulanish yopiladi
5. Mijoz ob'ektni qabul qiladi

**Kamchiliklari:**
- Ko'p TCP ulanishlar server resurslarini isrof qiladi.
- Har bir obyekt uchun **2 RTT** kechikish (TCP ulanish uchun 1 RTT + so'rov/javob uchun 1 RTT).
- **Misol**: HTML fayl + 10 rasm = 11 alohida ulanish.

##### 2.  ==Persistent== (Doimiy) 
- Bir TCP ulanish orqali bir nechta obyekt uzatiladi.
- Server javob yuborgandan keyin ulanishni ochiq qoldiradi
- **Konveyerlash (Pipelining)**: Bir nechta so'rovlarni javob kutmasdan uzatish.

#### HTTP Xabar Formatlari

##### So'rov Xabari (Request Message):
```
GET /somedir/page.html HTTP/1.1
Host: www.someschool.edu
Connection: close
User-agent: Mozilla/5.0
Accept-language: fr
```

**Tarkibi:**

1. **So'rov satri**: `GET /somedir/page.html HTTP/1.1`
    - Metod (GET)
    - URL yo'li
    - Protokol versiyasi
2. **Sarlavha satrlari**:
    - `Host:` - qaysi serverga murojaat
    - `Connection:` - ulanish turi (close/keep-alive)
    - `User-agent:` - brauzer turi
    - `Accept-language:` - til afzalligi

**HTTP Metodlari:**

- **GET** - ob'ektni olish (eng ko'p ishlatiladi)
- **POST** - forma ma'lumotlarini yuborish (parol, qidiruv so'zlari)
- **HEAD** - faqat sarlavhalarni olish (debugging uchun)
- **PUT** - faylni serverga yuklash
- **DELETE** - ob'ektni o'chirish
- **PATCH**
- **OPTIONS**
- **CONNECT**
- **TRACE**

##### Javob Xabari (Response Message)
```
HTTP/1.1 200 OK
Date: Tue, 09 Aug 2011 15:44:04 GMT
Server: Apache/2.2.3 (CentOS)
Last-Modified: Tue, 09 Aug 2011 15:11:03 GMT
Content-Length: 6821
Content-Type: text/html

(ma'lumotlar...)
```

**Tarkibi:**

1. **Holat satri**: `HTTP/1.1 200 OK`
2. **Sarlavhalar**
3. **Xabar tanasi** (so'ralgan ob'ekt)

**Muhim Holat Kodlari:**

- **200 OK** - muvaffaqiyatli, ob'ekt javobda
- **301 Moved Permanently** - ob'ekt boshqa joyga ko'chirilgan
- **304 Not Modified** - ob'ekt o'zgarmagan (kesh uchun)
- **400 Bad Request** - serverga tushunarsiz so'rov
- **404 Not Found** - ob'ekt topilmadi
- **505 HTTP Version Not Supported** - protokol versiyasi qo'llab-quvvatlanmaydi

## 6. Cookie Mexanizmi - Holatni Saqlash

**Muammo:** HTTP holatsiz protokol, lekin ba'zan foydalanuvchini kuzatish kerak.

**Cookie qanday ishlaydi:**

1. **Birinchi tashrif:**
    - Foydalanuvchi Amazon.com ga kiradi
    - Server unikal ID yaratadi (masalan: 1678)
    - Javobda `Set-cookie: 1678` yuboradi
    - Brauzer cookie-faylga saqlaydi
2. **Keyingi tashriflar:**
    - Brauzer har bir so'rovda `Cookie: 1678` yuboradi
    - Server foydalanuvchini taniydi

**Cookie komponenlari:**

1. Server javobidagi `Set-cookie:` sarlavhasi
2. Mijoz so'rovidagi `Cookie:` sarlavhasi
3. Brauzerdagi cookie-fayl
4. Serverdagi ma'lumotlar bazasi

**Foydalanish misollari:**

- Savat (shopping cart)
- Bir marta bosish bilan xarid
- Tavsiyalar tizimi
- Sessiya boshqaruvi



"HTTP TCP protokolini asosiy transport protokoli sifatida ishlatadi.  
Avval HTTP-klient server bilan TCP ulanishini boshlaydi. Ulanish o'rnatilgandan so'ng, brauzer va server protsesslari o'zlaridagi **soketlar** orqali TCP ga kirish huquqiga ega bo'ladi. Klient tomonda, soket – bu klient protsessi va TCP ulanishi o'rtasidagi **eshik**; server tomonda esa – server protsessi va TCP ulanishi o'rtasidagi eshik.

Klient HTTP-so'rovni soketi orqali yuboradi va HTTP-javobni shu soket orqali qabul qiladi. Xuddi shunday, HTTP-server so'rovni qabul qilib, javob xabarini o'z soketi orqali yuboradi. Klient xabarni soketga yuborgan zahoti, u TCP protokoli qo'liga o'tadi.
