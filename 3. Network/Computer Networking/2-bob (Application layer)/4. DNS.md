## **DNS (Domain Name System) - Internetning katalog xizmati**

### **1. Asosiy tushuncha: Nima uchun DNS kerak?**

DNS - bu ikki vazifani bajaradi:

- **Host nomlari** (masalan: [www.google.com](http://www.google.com)) - odamlar uchun qulay, esda qoluvchi
- **IP-addresslar** (masalan: 142.250.185.78) - kompyuterlar va routerlar uchun qulay, strukturalangan

DNS bu ikki identifikatsiya usulini bog'laydi, ya'ni host nomlarini IP-addresslarga tarjima qiladi.

### **2. DNS tizimining tarkibi - Taqsimlangan ierarxik tuzilma**

DNS markazlashtirilmagan, balki butun dunyo bo'ylab tarqalgan serverlardan iborat:

#### **DNS serverlari ierarxiyasi:**

1. **Root DNS serverlar** - eng yuqori daraja (13 ta, A dan M gacha belgilangan)
2. **TLD (Top-Level Domain) serverlar** - yuqori darajali domenlar uchun (.com, .org, .net, .ru, .uz)
3. **Authoritative DNS serverlar** - aniq tashkilotlar domenlari uchun mas'ul
4. **Local DNS server** - provayderning DNS serveri (default name server deb ham ataladi)

### **3. DNS so'rovlari turlari**

- **Recursive query** - server to'liq javobni topib berishi kerak
- **Iterative query** - server faqat keyingi qadamni ko'rsatadi

### **4. DNS Caching (Keshlash)**

DNS serverlar olgan javoblarni vaqtinchalik saqlab qo'yadi:

- Tezlikni oshiradi
- Tarmoq trafikini kamaytiradi
- TTL (Time To Live) - kesh yozuvining amal qilish muddati

### **5. DNS Resource Records (Resurs yozuvlari)**

Har bir yozuv 4 ta maydondan iborat: **(Name, Value, Type, TTL)**

| **Turi** | **Nomi**      | **Qiymati**                | **Izoh**                      |
| -------- | ------------- | -------------------------- | ----------------------------- |
| `A`      | Host nomi     | IP-manzil                  | Asosiy nom→IP bogʻlanishi     |
| `NS`     | Domen         | Avtoritativ server nomi    | Domen uchun javobgar server   |
| `CNAME`  | Psevdonim     | Kanonik nom                | Nomning taxallusi (alias)     |
| `MX`     | Pocht serveri | Kanonik pochta server nomi | Pochtaga ixtisoslangan server |
### **6. DNS xabarlar formati**

Ikki tur xabar mavjud:

- **DNS query** (so'rov)
- **DNS response** (javob)

Xabar tarkibi:

- **Header** (12 bayt) - identifikator, flaglar
- **Question section** - so'rov ma'lumotlari
- **Answer section** - javob yozuvlari
- **Authority section** - authoritative serverlar
- **Additional section** - qo'shimcha ma'lumot

### **7. DNS ishlash jarayoni (misol)**

Foydalanuvchi `www.amazon.com` saytiga kirmoqchi bo'lsa:

1. Browser local DNS serverga so'rov yuboradi
2. Local DNS → Root server (kim .com uchun javobgar?)
3. Root server → .com TLD server addressini qaytaradi
4. Local DNS → TLD server (kim amazon.com uchun javobgar?)
5. TLD server → amazon.com authoritative server addressi
6. Local DNS → Authoritative server ([www.amazon.com](http://www.amazon.com) IP-addressi?)
7. Authoritative server → IP-addressni qaytaradi
8. Browser TCP connection o'rnatadi va HTTP so'rov yuboradi

### **8. DNS xizmatlarining qo'shimcha vazifalari**

- **Host aliasing** - murakkab nomlarga qisqa taxalluslar berish
- **Mail server aliasing** - pochta serverlari uchun qulay nomlar
- **Load distribution** - yukni bir necha serverga taqsimlash (DNS rotation)

### **9. DNS xavfsizligi muammolari**

- **DDoS hujumlar** - serverlarni haddan tashqari so'rovlar bilan to'ldirish
- **DNS poisoning** - keshga soxta yozuvlar kiritish
- **MITM (Man-in-the-Middle)** - so'rovlarni ushlab, soxta javoblar berish
- **DNS reflection** - DNS infratuzilmasini boshqa hujumlar uchun ishlatish
