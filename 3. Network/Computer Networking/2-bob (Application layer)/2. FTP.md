### **FTP Protokolining Asosiy Xususiyatlari**

**1. FTP (File Transfer Protocol)**

- Fayllarni uzatish protokoli
- TCP asosida ishlaydi
- Holatni saqlaydi (stateful)
- Ikki parallel ulanishdan foydalanadi

### **Ikki Ulanish Arxitekturasi**

**2. Boshqaruv Ulanishi (Control Connection)**

```
Vazifasi:
- Port 21 da o'rnatiladi
- Butun seans davomida ochiq
- Buyruqlar va javoblar uzatiladi
- ASCII formatida ma'lumot

Uzatiladigan ma'lumotlar:
- USER/PASS (autentifikatsiya)
- LIST, RETR, STOR buyruqlari
- Katalog o'zgartirish buyruqlari
```

**3. Ma'lumotlar Ulanishi (Data Connection)**

```
Xususiyatlari:
- Har bir fayl uchun yangi ulanish
- Server tomonidan boshlanadi
- Fayl uzatilgandan keyin yopiladi
- Doimiy emas (non-persistent)
```

### **FTP vs HTTP Farqlari**

|Xususiyat|FTP|HTTP|
|---|---|---|
|Ulanishlar soni|2 (boshqaruv + ma'lumot)|1|
|Boshqaruv ma'lumoti|Out-of-band|In-band|
|Holat saqlash|Stateful|Stateless|
|Port|21 (boshqaruv)|80|

**4. Out-of-band vs In-band**

- **Out-of-band**: Boshqaruv va ma'lumot alohida kanallarda
- **In-band**: Boshqaruv va ma'lumot bitta kanalda

### **FTP Ishlash Jarayoni**

```
1. Ulanish o'rnatish:
   Mijoz → Server (port 21)
   
2. Autentifikatsiya:
   USER username
   PASS password
   
3. Fayl so'rash:
   RETR filename
   
4. Ma'lumot ulanishi:
   Server → Mijoz (yangi TCP)
   
5. Fayl uzatish:
   Fayl ma'lumotlari
   
6. Ma'lumot ulanishini yopish
   
7. Boshqaruv ulanishi ochiq qoladi
```

### **Asosiy FTP Buyruqlari**

**5. Autentifikatsiya buyruqlari:**

- `USER username` - foydalanuvchi nomi
- `PASS password` - parol

**6. Fayl operatsiyalari:**

- `LIST` - katalogdagi fayllar ro'yxati
- `RETR filename` - faylni yuklab olish (retrieve)
- `STOR filename` - faylni yuklash (store)
- `CWD directory` - katalogni o'zgartirish
- `PWD` - joriy katalogni ko'rsatish

### **FTP Server Javoblari**

**7. Javob kodlari tizimi:**

```
1xx - Ijobiy dastlabki javob
2xx - Ijobiy yakuniy javob
3xx - Ijobiy oraliq javob
4xx - Vaqtinchalik salbiy javob
5xx - Doimiy salbiy javob

Misollar:
331 - Parol kerak
125 - Uzatish boshlanmoqda
425 - Ma'lumot ulanishi ochilmadi
452 - Faylni yozishda xato
```

### **Holatni Saqlash (State Management)**

**8. FTP holatni saqlaydi:**

- Foydalanuvchi autentifikatsiyasi
- Joriy katalog pozitsiyasi
- Sessiya parametrlari
- Bu serverga yuklama yaratadi

**9. HTTP bilan taqqoslash:**

- HTTP holatsiz - har bir so'rov mustaqil
- FTP holatli - sessiya davomida ma'lumot saqlanadi

### **Afzalliklari va Kamchiliklari**

**Afzalliklari:**

- Katta fayllarni uzatish uchun optimallashtirilgan
- Uzatishni davom ettirish imkoniyati
- Ko'p fayllarni ketma-ket uzatish

**Kamchiliklari:**

- Murakkab (2 ulanish)
- NAT/Firewall muammolari
- Holatni saqlash serverni yuklaydi
- Xavfsizlik muammolari (matn formatida parol)

### **Zamonaviy Alternativalar**

- **SFTP/FTPS** - xavfsiz versiyalar
- **HTTP/HTTPS** - oddiy fayl uzatish uchun
- **Cloud storage API** - zamonaviy yechimlar
