# Trunk va 802.1Q

Trunk - bir nechta VLAN trafikini bitta fizik link orqali o'tkazadigan switchport rejimi. Agar SW1 va SW2 orasida VLAN 10, 20 va 99 ishlashi kerak bo'lsa, ular orasidagi link trunk bo'lishi kerak.

## 802.1Q nima?

802.1Q - Ethernet frame ichiga VLAN tag qo'shish standarti. Switch frame qaysi VLANga tegishli ekanini shu tag orqali biladi.

Oddiy tushuncha:

```text
Access portdan kelgan frame -> switch VLANni biladi
Trunkdan ketayotgan frame -> 802.1Q tag bilan ketadi
Trunkdan kelgan frame -> tagga qarab VLAN aniqlanadi
```

## Native VLAN

Native VLAN trunkda taglanmaydigan VLAN. Cisco qurilmalarda default native VLAN - VLAN 1. Xavfsizlik va tartib uchun native VLANni alohida, ishlatilmaydigan VLANga o'zgartirish tavsiya qilinadi.

```cisco
vlan 999
 name NATIVE-BLACKHOLE

interface gigabitEthernet0/1
 description TRUNK-TO-SW2
 switchport mode trunk
 switchport trunk native vlan 999
 switchport trunk allowed vlan 10,20,99,999
 no shutdown
```

Trunkning ikkala tomonida native VLAN bir xil bo'lishi kerak.

## Switch-switch trunk konfiguratsiyasi

SW1:

```cisco
configure terminal
vlan 10
 name USERS
vlan 20
 name SALES
vlan 99
 name MGMT

 description TRUNK-TO-SW2

end
```

```cisco
configure terminal

 name SALES
vlan 99

vlan 999

interface gigabitEthernet0/1
 description TRUNK-TO-SW1

end
```

## DTP haqida

DTP (Dynamic Trunking Protocol) Cisco switchlarda trunkni avtomatik kelishish uchun ishlatiladi. CCNA lablarda uchrashi mumkin, lekin productionda port rejimini qo'lda berish yaxshi.

```cisco
interface gigabitEthernet0/1
 switchport mode trunk
 switchport nonegotiate
```

Eslatma: `switchport nonegotiate` hamma platformalarda yoki Packet Tracerda doim ishlamasligi mumkin.

## Router-on-a-stick uchun trunk

Routerga bir nechta VLANni olib borish uchun switch-router link ham trunk bo'ladi:

```cisco
interface gigabitEthernet0/2
 description TRUNK-TO-R1
 switchport mode trunk
 switchport trunk allowed vlan 10,20
 no shutdown
```

Router tomonida subinterface sozlanadi. Bu `03-inter-vlan-routing.md` faylida tushuntirilgan.

## Tekshiruv komandalar

```cisco
show interfaces trunk
show interfaces gigabitEthernet0/1 switchport
show running-config interface gigabitEthernet0/1
show vlan brief
show interfaces status
```

`show interfaces trunk`da quyidagilarni tekshiring:

- Port trunk holatidami?
- Encapsulation 802.1Qmi?
- Native VLAN to'g'rimi?
- Allowed VLAN ro'yxatida kerakli VLAN bormi?
- STP forwarding holatidagi VLANlar qaysilar?

## Troubleshooting

Muammo: SW1 dagi VLAN 10 PC SW2 dagi VLAN 10 PCga ping qilmayapti.

```cisco
show vlan brief
show interfaces trunk
show mac address-table vlan 10
show spanning-tree vlan 10
```

Tekshiruv tartibi:

1. Ikkala switchda VLAN 10 yaratilganmi?
2. Uplink trunkmi?
3. Allowed VLAN ichida VLAN 10 bormi?
4. Native VLAN mismatch yo'qmi?
5. STP portni blocking qilmayaptimi?
6. PC IP/subnet to'g'rimi?

Muammo: Native VLAN mismatch.

```text
%CDP-4-NATIVE_VLAN_MISMATCH: Native VLAN mismatch discovered
```

Tuzatish:

```cisco
interface gigabitEthernet0/1
 switchport trunk native vlan 999
```

Ikkala tomonda ham bir xil qiling.

## Common mistakes

- Trunkning bir tomoni `access`, ikkinchi tomoni `trunk`.
- VLAN allowed listga qo'shilmagan.
- Native VLAN default VLAN 1 da qolgan.
- Native VLAN ikki tomonda farq qiladi.
- `switchport trunk allowed vlan 10` deb yozib, avvalgi VLANlarni tasodifan olib tashlash.
- Router-on-a-stickda switch port trunk emas.

Allowed VLANga qo'shishda ehtiyot bo'ling:

```cisco
switchport trunk allowed vlan add 30
switchport trunk allowed vlan remove 20
```

## Qisqa Q&A

**Savol:** Trunk bitta VLAN uchun ham kerak bo'ladimi?  
**Javob:** Agar faqat bitta VLAN kerak bo'lsa, access link yetarli. Trunk odatda bir nechta VLAN uchun.

**Savol:** 802.1Q tag access portga chiqadimi?  
**Javob:** Yo'q. Endpoint odatda tag ko'rmaydi. Switch access portda frame tagini olib tashlaydi.

**Savol:** Native VLANni allowed listga qo'shish kerakmi?  
**Javob:** Amaliyotda ha, trunkda native VLAN mos va boshqarilgan bo'lishi kerak.
